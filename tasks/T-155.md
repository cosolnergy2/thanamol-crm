# T-155: Fix Zone — can't use, can't save

**Status:** In Progress
**Assignee:** dev
**Created:** 2026-04-01
**Updated:** 2026-04-01

### Objective

Zone pages are broken: the project selector does not initialize properly, zone creation fails silently, and the edit page does not load existing data. This is a critical bug blocking all zone-dependent workflows.

Likely root causes to investigate:
- Project selector not initializing on page mount (empty state handling)
- Zone create form not wired to the correct API endpoint or missing required fields in the request body
- Zone edit page not fetching existing zone data before rendering the form

Files likely involved:
- `apps/web/src/routes/_authenticated/facility-management/zones/index.tsx`
- `apps/web/src/routes/_authenticated/facility-management/zones/create.tsx`
- `apps/web/src/routes/_authenticated/facility-management/zones/$zoneId/edit.tsx`
- `apps/web/src/hooks/useZones.ts`

### Detail

**Root causes identified and fixed:**

- **SelectItem empty string crash** (same issue as T-151): `zones/create.tsx` and `zones/$zoneId/edit.tsx` both passed `parentZoneIdValue ?? ''` as the `value` prop to Radix Select. Radix Select throws when a controlled `value=""` is set with no matching `SelectItem value=""`. Fixed by using `parentZoneIdValue || 'none'` so it always maps to the sentinel `'none'` item.

- **No auto-selection of project on list page**: `zones/index.tsx` used `useState('')` for `selectedProjectId` which never changed until the user manually picked a project. Added a `useEffect` that sets the first available project once projects load, so the zones list populates immediately on page visit.

- **API error format mismatch**: `apps/web/src/lib/api-client.ts` parsed error responses with `errorData.message` only, but the Elysia API returns `{ error: '...' }` format. All API errors (409 code conflict, 404 not found, 400 bad request) silently fell back to HTTP status text. Fixed by reading `errorData.message ?? errorData.error ?? statusText`.

**Files changed:**
- `apps/web/src/routes/_authenticated/facility-management/zones/index.tsx` — auto-select first project via useEffect
- `apps/web/src/routes/_authenticated/facility-management/zones/create.tsx` — fix SelectItem empty string crash
- `apps/web/src/routes/_authenticated/facility-management/zones/$zoneId/edit.tsx` — fix SelectItem empty string crash
- `apps/web/src/lib/api-client.ts` — handle `errorData.error` field in error responses
