### Objective
Build lead management and deal pipeline pages with exact UI parity to base44 reference.

Pages:
- /leads — LeadList with data table (pagination, search, filters by status/source/assignedTo)
- /leads/$leadId — LeadDetail with full info and activity timeline
- /deals — DealPipeline with Kanban board (drag-and-drop stage columns)
- /deals/$dealId — DealDetail with lead/customer relations

Acceptance criteria:
- All pages use TanStack Router file-based routing (no React Router)
- Data fetched via TanStack Query, integrated with T-020 API endpoints
- DealPipeline Kanban board supports drag-and-drop to move deals between stages
- LeadDetail shows activity timeline
- UI replicates base44 exactly: layout, column structure, color coding for stages
- All shared types from packages/shared/src/types/
- No hardcoded colors — use indigo/teal Tailwind design tokens

Reference base44: LeadList.jsx, LeadDetail.jsx, DealPipeline.jsx, DealDetail.jsx
CRITICAL: Replicate exact UI from base44.

### Detail

**Implementation approach:**
- Create `useLeads.ts` and `useDeals.ts` hooks mirroring existing hook patterns (TanStack Query + apiGet/apiPost/apiPut/apiDelete)
- Lead status uses system values: NEW, CONTACTED, QUALIFIED, UNQUALIFIED, CONVERTED
- Deal stages use system values: PROSPECTING, QUALIFICATION, PROPOSAL, NEGOTIATION, CLOSED_WON, CLOSED_LOST
- DealPipeline uses click-to-move (select dropdown in card) instead of drag-and-drop since @hello-pangea/dnd is not installed; loads all deals via `limit=100` to GET /api/deals
- Convert-to-deal on LeadDetail creates a new Deal with leadId set and updates lead status to CONVERTED
- All pages use TanStack Router `createFileRoute`, no React Router
