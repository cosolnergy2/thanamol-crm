# T-090: Replace inventory photo URL input with file upload

**Status:** In Progress
**Assignee:** dev
**Created:** 2026-04-07
**Updated:** 2026-04-07

### Objective

The inventory create and edit forms currently use a plain text input for the photo field, requiring users to manually type or paste a URL. All other forms in the system use the existing `FileUpload` component for image fields. This inconsistency prevents users from uploading photos directly.

Fix by replacing the text URL input with the `FileUpload` component in both the inventory create and edit forms.

Acceptance criteria:
- Inventory create form photo field uses `FileUpload` component (not a text input)
- Inventory edit form photo field uses `FileUpload` component and pre-populates with the existing photo if present
- Uploaded file is stored and the resulting URL is saved to the inventory record, consistent with how other forms handle file uploads
- `pnpm build` passes with zero errors
- `pnpm test` passes

### Detail

**Implementation approach:**
- Add `import { FileUpload } from '@/components/FileUpload'` to both files.
- Replace the `<Label htmlFor="photoUrl">Image URL</Label>` + `<Input id="photoUrl" ...>` block with `<FileUpload label="Photo" accept="image/*" value={form.photoUrl || undefined} onChange={(url) => setForm({ ...form, photoUrl: url ?? '' })} />`.
- The edit form already initialises `form.photoUrl` from the existing record via `useEffect`, so pre-population requires no additional changes.
- The API payload shape (`photos: form.photoUrl ? [{ url: form.photoUrl }] : undefined`) remains unchanged — `FileUpload` sets `form.photoUrl` to the returned server URL, which slots straight into the existing submit logic.
- `Input` import is retained in both files — used by all other fields.

**Files changed:**
- `apps/web/src/routes/_authenticated/facility-management/inventory/create.tsx`
- `apps/web/src/routes/_authenticated/facility-management/inventory/$itemId/edit.tsx`

**Reference:**
- `apps/web/src/components/FileUpload.tsx` — component interface: `{ value?: string; onChange: (url: string | undefined) => void; accept?: string; label?: string }`
- Usage pattern from `apps/web/src/routes/_authenticated/facility-management/vendors/invoices/create.tsx`
