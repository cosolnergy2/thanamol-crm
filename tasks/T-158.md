# T-158: Asset detail/edit page enhancement

**Status:** In Progress
**Assignee:** dev
**Created:** 2026-04-01
**Updated:** 2026-04-01

### Objective

Update the Asset detail and edit pages to display and allow editing of all new fields added in T-157. The detail page must render all new fields (scope type, criticality, lifecycle status, condition score, brand, supplier, dates, specifications, notes). The edit form must pre-fill with existing data and save changes correctly.

Acceptance criteria:
- Detail page shows all new fields, gracefully handling null/empty values
- Edit form pre-fills all fields from the existing asset record
- Saving the edit form persists changes and the list reflects updated data

### Detail

**Implementation approach:**
- Updated `$assetId/index.tsx` to display all new fields organized in four cards: Basic Information (category, criticality badge, condition score, lifecycle status badge, description), Ownership & Location (scope type, project, zone, unit, location detail, assignee), Technical Details (brand, manufacturer, model, serial, supplier resolved via `useVendor`), and Purchase & Warranty (price, purchase date, install date, warranty end). Added an Additional Information card that reads `notes` from `specifications.notes` and renders any other specifications key/value pairs.
- Updated `$assetId/edit.tsx` to mirror the create form exactly: added scopeType, criticality, conditionScore, lifecycleStatus, brand, supplierId (vendor select), installDate, notes, imageUrl fields with pre-fill from existing asset data.
- Pre-fill logic uses `useEffect` on `asset` data load; `isoDateString` helper converts server datetime strings to YYYY-MM-DD for date inputs; `extractNotes` reads `specifications.notes` from the JSON blob.
- Supplier displayed in detail page by resolving `supplier_id` via `useVendor` hook (lazy fetch only when `supplier_id` is set).
- `__none__` sentinel values used in Select components to clear optional FK fields on save.
