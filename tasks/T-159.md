# T-159: Work Order API + form enhancement

**Status:** Ready to Dev
**Assignee:** dev
**Created:** 2026-04-01
**Updated:** 2026-04-01

### Objective

Enhance the Work Order API and create/detail forms to expose all new schema fields added in T-152. Users currently cannot set Budget Code, Scheduled Start/End, or assign a Vendor when creating a Work Order. The form must also display an auto-generated WO Number as a read-only field.

Acceptance criteria:
- API create and update schemas accept `budget_code`, `scheduled_start`, `scheduled_end`, `vendor_id`
- Create form shows WO Number (auto-generated, read-only), Budget Code input, Scheduled Start/End datetime pickers, Technician dropdown, Vendor dropdown
- WO Type dropdown includes: Corrective, Preventive, Emergency, Inspection, Calibration
- Priority dropdown includes: Low, Medium, High, Urgent
- Detail page displays all new fields
- All values sourced from constants in `packages/shared` — no hardcoded strings
