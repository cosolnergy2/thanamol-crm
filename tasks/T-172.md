# T-172: Vendor detail/edit enhancement

**Status:** Ready to Test
**Assignee:** dev
**Created:** 2026-04-01
**Updated:** 2026-04-01

### Objective
Enhance the vendor detail page to use a tabbed layout that surfaces all related records without leaving the vendor context. Currently the vendor detail page shows only basic info. It needs to be reorganised into tabs and wired to the additional fields introduced in T-171.

Acceptance criteria:
- Vendor detail page has tabs: Info, Contracts, Invoices, Item Prices
- Info tab displays all fields from the overhauled registration form (T-171): company details, contact info, additional contacts list, service tags, payment terms, default conditions
- Contracts tab lists vendor contracts associated with this vendor (links to T-173 work)
- Invoices tab lists vendor invoices associated with this vendor (links to T-174 work)
- Edit form mirrors the create form from T-171 — all fields pre-filled from existing data
- Saving edits persists changes correctly (e.g. updating rating, status, payment terms)

### Detail

**Implementation approach:**
- Rewrote `$vendorId/index.tsx`: added `info` tab as default, replaced old 2-card layout with single tabbed Card; added `InfoTab` component showing all T-171 fields (legal name, display name, vendor type, tax ID, additional contacts, service tags, supplier type, payment terms, credit limit, default conditions, notes)
- Rewrote `$vendorId/edit.tsx`: mirrors create.tsx with sections for ข้อมูลพื้นฐาน, ชื่อผู้ติดต่อหลัก, ผู้ติดต่อสำรอง, Tags, การจัดหมู่ Supplier, เงื่อนไขการชำระเงิน, Default Conditions, หมายเหตุ; pre-fills all fields from vendorData via useEffect
- Used `VendorWithExtras` local type to allow accessing JSON fields (service_tags, additional_contacts, default_conditions) from the Vendor type
- Exported `VendorDetailPage` and `VendorEditPage` to enable unit testing
