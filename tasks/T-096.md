# T-096: Incident Report — add missing fields + fix locationDetail rendering

**Status:** In Progress
**Assignee:** dev
**Created:** 2026-04-07
**Updated:** 2026-04-07

### Objective

Two separate problems on the Incident Report create form, both fixed in this task:

**1. Missing fields (full-stack):** The `Incident` Prisma model is missing fields present in the Base44 reference:
- `incident_type` — select with options: Safety, Security, Fire, Environmental, Property Damage, Injury, Near Miss, Other
- `vendor_involved` — text or relation reference
- `site_id` — relation or text reference to site

**2. locationDetail rendering bug (front-end only):** `locationDetail` exists in form state but the corresponding `<Input>` element was never added to the JSX. The field is silently dropped on every submission.

Acceptance criteria:
- `incident_type`, `vendor_involved`, `site_id` added to `Incident` Prisma model with appropriate types and defaults
- A Prisma migration is created using `prisma migrate dev` (not `db push`)
- API create and update routes accept and persist all new fields
- Shared `Incident` type and `CreateIncidentRequest` updated to include new fields
- Incident create form renders:
  - `incident_type` as a select/dropdown with all 8 type options
  - `vendor_involved` as a text input
  - `site_id` as a dropdown or text input
  - `locationDetail` as a visible text input (the missing element, wired to existing state)
- `pnpm build` passes with zero errors
- `pnpm test` passes

Files to change:
- `apps/api/prisma/schema.prisma`
- Incident compliance route (under `apps/api/src/routes/fms/compliance.ts` or dedicated route file)
- `packages/shared/src/types/index.ts`
- `apps/web/src/routes/_authenticated/facility-management/compliance/incidents/create.tsx`
