# T-093: PM Compliance Report — add missing features

**Status:** In Progress
**Assignee:** dev
**Created:** 2026-04-07
**Updated:** 2026-04-07

### Objective

The PM Compliance Report page is missing several features present in the Base44 reference:

1. Time period filter dropdown — options: Last 30 Days, Last 60 Days, Last 90 Days, Last 6 Months
2. "Completed" stat card (currently absent from the summary row)
3. PM Schedule Details table — full column set: Site, Frequency, Total, Completed, Overdue, Scheduled, Compliance %

This requires changes across the full stack: API query filtering by period, shared type updates, query hook, and UI.

Acceptance criteria:
- API accepts a `period` query parameter (`30d` | `60d` | `90d` | `6m`) and filters PM data accordingly
- Shared `PMComplianceReport` type includes a `completed` count field and all table column fields
- `usePMComplianceReport` hook passes the selected period to the API
- UI renders the period dropdown wired to hook state
- Stat card row includes a "Completed" card alongside existing cards
- PM Schedule Details table renders all columns: Site, Frequency, Total, Completed, Overdue, Scheduled, Compliance %
- Layout matches Base44 reference
- `pnpm build` passes with zero errors
- `pnpm test` passes

Files to change:
- `apps/api/src/routes/fms/reports.ts`
- `apps/web/src/hooks/useFMSReports.ts`
- `apps/web/src/routes/_authenticated/facility-management/reports/pm-compliance.tsx`
- `packages/shared/src/types/index.ts`

### Detail

Implementation approach:
- Add `buildPMComplianceReport(projectId, period?)` to `apps/api/src/routes/fms/reports.ts` — queries `PreventiveMaintenance` with optional date window derived from `period` param, uses `PMScheduleLog` to compute completed/overdue/scheduled counts per schedule, returns `completedCount` and enhanced `scheduleDetails` rows (site, frequency, total, completed, overdue, scheduled, compliance)
- Add `GET /api/fms/reports/pm-compliance` route accepting `projectId` + `period` (`30d`|`60d`|`90d`|`6m`) query params
- Update `PMComplianceReport` shared type to include `completedCount` and `scheduleDetails` array
- Update `FMS_REPORTS_QUERY_KEYS.pmCompliance` to include `period`; update `usePMComplianceReport` to accept and forward `period`
- Replace UI "Overdue PM Schedules" table with "PM Schedule Details" full-column table; add Completed stat card; add period dropdown filter next to project selector
