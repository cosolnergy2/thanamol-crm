# T-025: Frontend — Commercial quotation pages

**Status:** In Progress
**Assignee:** dev
**Created:** 2026-03-15
**Updated:** 2026-03-15

### Objective
Build commercial quotation CRUD pages with exact UI parity to base44 reference.

Pages:
- /quotations/commercial/new — NewCommercialQuotation (create form)
- /quotations/commercial/$id/edit — EditCommercialQuotation (edit form)
- /quotations/commercial/$id — ViewCommercialQuotation (detail view with approve/reject)
- /quotations/commercial/$id/print — PrintCommercialQuotation (print-optimized layout)

Acceptance criteria:
- All pages use TanStack Router file-based routing
- Data fetched via TanStack Query integrated with T-023 /api/commercial-quotations endpoints
- Approve/reject actions gated by manage_contracts permission via PermissionGuard
- PrintCommercialQuotation hides sidebar/nav, optimized for browser print dialog
- UI replicates base44 exactly
- No hardcoded colors — use indigo/teal Tailwind design tokens

Reference base44: NewCommercialQuotation.jsx, EditCommercialQuotation.jsx, ViewCommercialQuotation.jsx, PrintCommercialQuotation.jsx

### Detail

**Implementation approach:**
- Add shared types `RejectCommercialQuotationRequest` to `packages/shared/src/types/index.ts`
- Build `useCommercialQuotations.ts` hook covering list, single get, create, update, delete, approve, reject mutations
- Build `CommercialQuotationForm` component for reuse across new/edit pages (customer selector, project selector, items table, terms/conditions)
- Build four route pages using TanStack Router `createFileRoute` with `$id` path param
- Print page hides sidebar/nav via `print:hidden` and renders a clean A4-optimized layout inline

**API endpoints used:**
- `GET /api/commercial-quotations` — list with pagination/filters
- `GET /api/commercial-quotations/:id` — single quotation with relations
- `POST /api/commercial-quotations` — create
- `PUT /api/commercial-quotations/:id` — update
- `DELETE /api/commercial-quotations/:id` — delete
- `POST /api/commercial-quotations/:id/approve` — approve (requires manage_contracts)
- `POST /api/commercial-quotations/:id/reject` — reject with reason (requires manage_contracts)
