### Objective

Initialize the full monorepo source structure so that `pnpm install`, `pnpm build`, and `pnpm dev` all work end-to-end with zero TypeScript errors. This is the foundation every other task builds on.

**Files to create:**

Root level:
- `package.json` — workspace script shortcuts (`dev`, `build`, `test`)
- `pnpm-workspace.yaml` — packages: `apps/*`, `packages/*`
- `.gitignore` — updated with `node_modules`, `dist`, `.env`, Playwright report dirs, etc.

`apps/api/`:
- `package.json` — runtime deps: `elysia`, `@elysiajs/cors`, `@elysiajs/jwt`, `@elysiajs/static`, `@prisma/client`, `prisma`; devDeps: `bun-types`, `typescript`, `vitest`
- `tsconfig.json`
- `src/index.ts` — Elysia hello-world with CORS plugin

`apps/web/`:
- `package.json` — runtime deps: `react`, `react-dom`, `@tanstack/react-router`, `@tanstack/router-devtools`, `@tanstack/react-query`, `tailwindcss`, `@tailwindcss/vite`, `@radix-ui/react-dialog`, `@radix-ui/react-dropdown-menu`, `@radix-ui/react-select`, `@radix-ui/react-tabs`, `@radix-ui/react-tooltip`, `@radix-ui/react-popover`, `@radix-ui/react-checkbox`, `@radix-ui/react-label`, `@radix-ui/react-separator`, `@radix-ui/react-slot`, `recharts`, `react-hook-form`, `lucide-react`, `framer-motion`, `date-fns`, `jspdf`, `html2canvas`, `react-leaflet`, `sonner`, `class-variance-authority`, `clsx`, `tailwind-merge`; devDeps: `@vitejs/plugin-react`, `vite`, `typescript`, `@tanstack/router-plugin`, `@types/react`, `@types/react-dom`, `playwright`, `@playwright/test`
- `vite.config.ts` — uses `@vitejs/plugin-react` and `@tanstack/router-plugin/vite`
- `tsconfig.json`
- `tailwind.config.ts` — indigo/teal design tokens with CSS variables for dark mode, matching base44 palette
- `src/main.tsx` — React 19 entry point with `RouterProvider`
- `src/routeTree.gen.ts` — TanStack Router generated tree placeholder (auto-generated by router plugin)
- `index.html` — Vite entry HTML
- `playwright.config.ts` — base URL `http://localhost:5173`

`packages/shared/`:
- `package.json` — name: `@thanamol/shared`, with `exports` pointing to `src/index.ts`
- `tsconfig.json`
- `src/index.ts` — barrel export placeholder

**Tailwind design tokens (indigo/teal palette):**
- Primary: indigo-600 / indigo-700
- Secondary: teal-500 / teal-600
- Background: white / gray-50
- Text: gray-900 / gray-600
- CSS variables for dark mode via `darkMode: 'class'`

**Acceptance criteria:**
1. `pnpm install` succeeds with no errors
2. `pnpm build` compiles all 3 packages with zero TypeScript errors
3. `pnpm dev` starts API on port 3000 and web on port 5173
4. Navigating to `http://localhost:5173` renders the app without a blank screen

### Detail

**Implementation approach:**
- Scaffold pnpm workspaces at root with `pnpm-workspace.yaml` pointing to `apps/*` and `packages/*`
- Build `packages/shared` as a simple TypeScript barrel package with `src/index.ts` and placeholder types
- Build `apps/api` as an Elysia.js app with Bun runtime: CORS plugin, health endpoint at `/api/health`
- Build `apps/web` as a Vite + React 19 + TanStack Router app with Tailwind v4 (CSS-based config via `@theme` block in index.css, no JS config file needed)
- Port the indigo/teal CSS variable design tokens from base44 `index.css` into `apps/web/src/index.css` using Tailwind v4's `@theme` syntax
