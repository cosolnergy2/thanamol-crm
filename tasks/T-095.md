# T-095: Permit to Work — add missing fields, Workers, PPE

**Status:** In Progress
**Assignee:** dev
**Created:** 2026-04-07
**Updated:** 2026-04-07

### Objective

The Permit to Work create form is missing several fields and two complex sections (Workers and PPE Required) that exist in the Base44 reference. This requires full-stack changes.

Missing scalar fields to add to `PermitToWork` Prisma model:
- `company_id` — relation or text reference to company
- `site_id` — relation or text reference to site
- `location` — text
- `unit` — text
- `contractor_contact` — text

Missing JSON fields to add to `PermitToWork` Prisma model:
- `workers` — JSON array, each entry has: `name` (string), `id_number` (string)
- `ppe_required` — JSON array of selected PPE item strings

PPE options (fixed list for checkboxes): Hard Hat, Safety Glasses, High Vis Vest, Gloves, Safety Shoes, Ear Protection, Respirator, Fall Protection

Acceptance criteria:
- All scalar fields added to `PermitToWork` Prisma model
- `workers Json @default("[]")` and `ppe_required Json @default("[]")` added to model
- A Prisma migration is created using `prisma migrate dev` (not `db push`)
- API create and update routes accept and persist all new fields
- Shared `PermitToWork` type and `CreatePermitToWorkRequest` updated
- Permit create form includes:
  - All new scalar fields as text inputs / dropdowns
  - Workers section: add/remove rows, each row has Name + ID Number inputs
  - PPE Required section: grid of checkboxes for the 8 PPE options
- `pnpm build` passes with zero errors
- `pnpm test` passes

Files to change:
- `apps/api/prisma/schema.prisma`
- Permit to Work compliance route (under `apps/api/src/routes/fms/compliance.ts` or dedicated route file)
- `packages/shared/src/types/index.ts`
- `apps/web/src/routes/_authenticated/facility-management/compliance/permits/create.tsx`

### Detail
