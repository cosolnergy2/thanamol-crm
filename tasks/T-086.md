# T-086: Unit form — add 15+ missing fields

**Status:** In Progress
**Assignee:** dev
**Priority:** HIGH

## Objective

Add 15+ missing fields to the Unit form that exist in the Base44 reference design but are missing from the current implementation. Fields span two categories: Technical/Physical specs and Pricing.

## Acceptance Criteria

- [ ] Unit form has a "Location & Physical" section with: Zone, Location, Office Area sqm, Ceiling Height, Floor Load, Electrical Load, Has Sprinkler, Lease Type, Floor Plan URL
- [ ] Unit form has a "Pricing" section with: Rent/sqm, Common Fee + waived toggle, Water Rate + actual toggle, Electricity Rate + actual toggle, Deposit Months, Advance Rent Months
- [ ] Toggle pattern: when "waived/actual" checkbox is checked, the number input is disabled
- [ ] All new fields persist to the database on create and update
- [ ] `pnpm build` passes with zero errors
- [ ] `pnpm test` passes

## Definition of Done

- Prisma schema updated with 17 new columns on Unit model
- Migration created and applied
- Shared types updated (Unit, CreateUnitRequest, UpdateUnitRequest)
- LEASE_TYPES constant added to packages/shared
- API create/update schemas include all new fields
- Frontend form expanded into 3 sections with toggle pattern
