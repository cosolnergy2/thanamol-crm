### Objective

Add missing fields to Procurement and Vendor models. These changes enable the full PR/PO form overhaul in Phase 5 and Vendor management overhaul in Phase 6.

Fields to add:
- **PurchaseRequest:** `company_id`, `site_id`, `unit_id`, `required_date`, `purpose`, `pm_schedule_id`, `documents` (Json), `conditions` (Json)
- **PurchaseOrder:** `company_id`, `site_id`, `unit_id`, `po_date`, `payment_due_date`, `po_type`, `delivery_address`, `documents` (Json), `conditions` (Json)
- **Vendor:** `legal_name`, `display_name`, `vendor_type`, `company_registration`, `additional_contacts` (Json), `service_tags` (Json), `supplier_type`, `payment_terms`, `credit_limit`, `default_conditions` (Json)
- **VendorContract:** `contract_type`, `service_category`, `sla` (Json), `rate_card` (Json), `alert_days_before_expiry`, `auto_renew`
- **VendorInvoice:** `pdf_url`, `submission_history` (Json)

Also add new shared constants: `PR_PURPOSES`, `PR_ITEM_TYPES`, `PO_TYPES`, `VENDOR_TYPES`, `VENDOR_SERVICE_TAGS`, `VENDOR_CONTRACT_TYPES`.

Acceptance criteria: migration runs cleanly, build passes, no existing tests break.
