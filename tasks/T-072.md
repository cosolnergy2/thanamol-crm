# T-072: Invoice E2E tests

**Status:** Ready to Dev
**Assignee:** dev
**Created:** 2026-03-19
**Updated:** 2026-03-19

### Objective

Write Playwright E2E tests covering the enhanced invoice feature implemented in T-070 and T-071. Tests should run against the full stack (API + frontend).

#### Scope

Create `apps/web/e2e/invoices.spec.ts` covering:

1. **Invoice list page**
   - Page loads and displays the invoice table.
   - Stats cards (All, Outstanding, Partial, Paid, Overdue) are visible.
   - Status filter dropdown filters the list correctly.
   - Search by invoice number narrows results.
   - Search by customer name narrows results.

2. **Invoice create**
   - Navigate to `/finance/invoices/create`.
   - Select a customer.
   - Set invoice date and due date.
   - Set billing period start and end.
   - Add a line item via dialog (with item type = "Rent").
   - Verify discount field updates the total.
   - Submit form; expect redirect to invoice list and new invoice visible in list.

3. **Invoice detail**
   - Navigate to a created invoice's detail page.
   - Invoice date, billing period, and line item type are displayed.
   - "Mark as Sent" button changes status to SENT.
   - "Record Payment" button opens dialog; fill in amount and method; submit; payment history shows the new entry; invoice status updates to PAID or PARTIAL.
   - "Void Invoice" button opens confirmation dialog; confirm; status changes to CANCELLED.

4. **Invoice edit**
   - Navigate to a DRAFT invoice's edit page.
   - Change the due date and add a second line item.
   - Save; verify changes are reflected on the detail page.

5. **Status transition guard**
   - A PAID or CANCELLED invoice's edit link is not accessible (redirects or shows "Only DRAFT invoices can be edited").

6. **Delete guard**
   - Delete button is only visible on DRAFT invoices in the list.

#### Acceptance Criteria

- All 6 scenario groups pass with 0 failures.
- Tests are committed on the topic branch.
- `pnpm --filter web exec playwright test e2e/invoices.spec.ts` exits 0.
