# T-075 Role Management Enhancement

**Status:** Test Pass
**Assignee:** tester

## Objective

Enhance the role management system with granular permission model, system roles, role templates, and a matrix permission UI.

## Detail

### Implementation Approach
- Add `PERMISSION_MODULES`, `PERMISSION_ACTIONS`, `ROLE_TEMPLATES`, and legacy key mapping to `packages/shared/src/constants/permissions.ts`
- Update Prisma `Role` model with `code` and `is_system_role` fields; generate and apply migration
- Update `apps/api/src/middleware/permissions.ts` to handle both flat and nested permission formats, flatten nested → dot-notation + legacy keys
- Update `apps/api/src/routes/roles.ts` with user_count on GET, code field on create/update, system role guards, and GET /roles/templates
- Update `apps/web/src/routes/_authenticated/settings/roles.tsx` with matrix grid UI, system role lock, user count badge, template pre-fill
- Add `hasModulePermission(module, action)` to `apps/web/src/hooks/usePermissions.ts`
- Update `apps/api/prisma/seed.ts` with nested permissions, code, is_system_role for all roles

### Files to Create or Modify
- `packages/shared/src/constants/permissions.ts` — new file
- `packages/shared/src/index.ts` — export constants
- `apps/api/prisma/schema.prisma` — Role model: add code, is_system_role
- `apps/api/prisma/migrations/...` — migration for new fields
- `apps/api/src/middleware/permissions.ts` — handle flat+nested, add hasModulePermission
- `apps/api/src/routes/roles.ts` — enhancements
- `apps/web/src/routes/_authenticated/settings/roles.tsx` — UI rewrite
- `apps/web/src/hooks/usePermissions.ts` — add hasModulePermission
- `apps/api/prisma/seed.ts` — nested permissions + new roles

### API Endpoints
- `GET /api/roles` — returns roles with `user_count`
- `POST /api/roles` — accepts `code`, nested permissions
- `PUT /api/roles/:id` — blocks name/code change for system roles; accepts nested permissions
- `DELETE /api/roles/:id` — blocks if `is_system_role`
- `GET /api/roles/templates` — returns role templates from shared constants

### Database Model Changes
- `Role`: add `code String? @unique`, `is_system_role Boolean @default(false)`
