# T-024: Frontend — QuotationList/Create/Edit/Detail, PendingApproval, PrintQuotation

**Status:** In Progress
**Assignee:** dev
**Created:** 2026-03-15
**Updated:** 2026-03-15

### Objective
Build all standard quotation pages with exact UI parity to base44 reference.

Pages:
- /quotations — QuotationList (table with filters by status/customer/project, pagination)
- /quotations/create — QuotationCreate form
- /quotations/$quotationId — QuotationDetail (with approve/reject actions for permitted users)
- /quotations/$quotationId/edit — QuotationEdit form
- /quotations/pending — QuotationPendingApproval (list of quotations awaiting approval)
- /quotations/$quotationId/print — PrintQuotation (print-optimized layout, no sidebar/nav)

Acceptance criteria:
- All pages use TanStack Router file-based routing
- Data fetched via TanStack Query integrated with T-023 API endpoints
- Approve/reject buttons on detail and pending pages only visible to users with manage_contracts permission (use PermissionGuard)
- PrintQuotation hides sidebar/nav and is optimized for browser print dialog
- UI replicates base44 exactly
- No hardcoded colors — use indigo/teal Tailwind design tokens

Reference base44: QuotationList.jsx, QuotationCreate.jsx, QuotationDetail.jsx, QuotationEdit.jsx, QuotationPendingApproval.jsx, PrintQuotation.jsx

### Detail

**Implementation approach:**
- Create `useQuotations.ts` hook with CRUD + approve/reject/pending queries using TanStack Query + `apiGet/apiPost/apiPut/apiDelete`
- Build 6 TanStack Router file-based routes under `_authenticated/quotations/`
- QuotationList: table with status/customer/project filters, status badges, action column (view, edit, delete)
- QuotationCreate/Edit: single form with line items (items JSON array), customer/project/unit selectors, discount/tax/total calculation
- QuotationDetail: full view with status badge, line items table, approve/reject buttons behind `PermissionGuard permission="manage_contracts"`
- QuotationPendingApproval: card list of SENT quotations with inline approve/reject dialogs
- PrintQuotation: clean layout with `print:hidden` nav bar, company header, line items, totals — uses `window.print()`

**API endpoints used (T-023):**
- `GET /api/quotations` — list with filters (search, status, customerId, projectId, page, limit)
- `GET /api/quotations/pending` — SENT quotations
- `GET /api/quotations/:id` — detail with relations
- `POST /api/quotations` — create
- `PUT /api/quotations/:id` — update
- `DELETE /api/quotations/:id` — delete
- `POST /api/quotations/:id/approve` — approve (requires manage_contracts)
- `POST /api/quotations/:id/reject` — reject with reason (requires manage_contracts)
