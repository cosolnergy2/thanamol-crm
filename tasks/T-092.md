# T-092: Fix Inventory Analysis SelectItem empty value crash

**Status:** In Progress
**Assignee:** dev
**Created:** 2026-04-07
**Updated:** 2026-04-07

### Objective

The Inventory Analysis page crashes on load because a Radix UI `<SelectItem>` has `value=""` (empty string), which Radix Select does not allow — it treats an empty string as no selection and throws an error.

Fix is contained to a single file. Change the empty-value SelectItem to use `"all"` as its value, align the state initializer, and update the filtering logic to match.

Acceptance criteria:
- `apps/web/src/routes/_authenticated/facility-management/reports/inventory-analysis.tsx` line 144: `<SelectItem value="">` changed to `<SelectItem value="all">`
- The corresponding `useState('')` initializer changed to `useState('all')`
- Filtering logic updated so `value === "all"` means "show everything" (equivalent to the previous empty-string check)
- Inventory Analysis page loads without any crash or console error
- `pnpm build` passes with zero errors
- `pnpm test` passes

### Detail

Implementation approach:
- Change `useState('')` to `useState('all')` on line 92 so the Select has a valid initial value.
- Change `<SelectItem value="">` to `<SelectItem value="all">` on line 144 to satisfy Radix Select's constraint that item values must not be empty strings.
- Change `projectId || undefined` to `projectId === 'all' ? undefined : projectId` on line 95 so the "all projects" sentinel correctly maps to an undefined API parameter rather than passing the string `"all"` as a project ID.

Files changed:
- `apps/web/src/routes/_authenticated/facility-management/reports/inventory-analysis.tsx`
