# T-074: User Management Enhancement

**Status:** Testing
**Assignee:** tester
**Branch:** `feat/T-074-user-management-enhancement`

## Objective

Enhance the user management system with additional profile fields, role assignment on creation, edit dialog, reset password functionality, and department filtering.

## Detail

### Implementation Approach
- Added `phone`, `department`, `position` fields to Prisma `User` model
- Added `DEPARTMENTS` constant and new types (`CreateUserRequest`, `ResetPasswordRequest`) to shared package
- Updated `AuthUser` and `AuthenticatedUser` types to include the new profile fields
- Enhanced `POST /auth/register` to accept and store new fields, assign role in transaction
- Enhanced `PUT /auth/users/:id` to accept new fields and replace role assignment in transaction
- Added `POST /auth/users/:id/reset-password` endpoint
- Updated `GET /auth/users` to return new profile fields
- Rewrote the frontend users page with: create dialog (with new fields + role), edit dialog (click row), reset password dialog, department filter, improved user row display

### Files Modified
- `apps/api/prisma/schema.prisma` — Added `phone`, `department`, `position` to User model
- `packages/shared/src/types/index.ts` — Added `DEPARTMENTS`, updated `AuthUser`, added `CreateUserRequest`, `ResetPasswordRequest`
- `apps/api/src/middleware/auth.ts` — Updated `AuthenticatedUser` type with new fields
- `apps/api/src/routes/auth.ts` — Enhanced register, PUT, GET /users; added reset-password endpoint
- `apps/api/src/routes/auth.test.ts` — Updated mocks and tests for new endpoints
- `apps/web/src/routes/_authenticated/settings/users.tsx` — Full UI overhaul with edit/reset dialogs and department filter
- `apps/web/src/components/layout/Sidebar.test.tsx` — Updated AuthUser mock to include new fields

### API Endpoints
- `POST /api/auth/register` — now accepts `phone?`, `department?`, `position?`, `roleId?`
- `PUT /api/auth/users/:id` — now accepts `phone?`, `department?`, `position?`, `roleId?` (replaces current role)
- `POST /api/auth/users/:id/reset-password` — new; body `{ newPassword: string }`
- `GET /api/auth/users` — now returns `phone`, `department`, `position` per user
