# T-073: Register All Missing API Routes

**Status:** In Progress
**Assignee:** dev
**Created:** 2026-03-19
**Updated:** 2026-03-19

### Objective
Several route files exist in `apps/api/src/` but are never mounted in `apps/api/src/index.ts`, meaning the endpoints they define return 404 in production. The goal is to find all 17 unregistered route files and mount them in the main app entry point so every endpoint in the codebase is reachable.

Acceptance criteria:
- All route files under `apps/api/src/` are imported and registered in `apps/api/src/index.ts`.
- No existing registered routes are removed or reordered in a breaking way.
- `pnpm build` passes with no TypeScript errors.
- A smoke-test GET to each newly registered route base path returns something other than 404.

### Detail
- Audited all files under `apps/api/src/routes/` and compared against imports in `apps/api/src/index.ts`.
- Found 17 route exports not yet mounted: activity-logs, automation-rules, client-portal, comments, deposits, handover-photos, handovers, lease-agreements, meter-records, notifications (x2), pre-handover-inspections, sale-jobs, task-config, task-statuses, tasks, tickets, warehouse-requirements.
- Added missing imports and `.use()` calls to `apps/api/src/index.ts` preserving all existing registrations and order.

Files changed:
- `apps/api/src/index.ts` — added 17 imports and `.use()` calls
- `tasks/T-073.md` — status updated to In Progress
