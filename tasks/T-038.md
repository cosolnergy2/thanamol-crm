**Status:** Ready to Test

**Updated:** 2026-03-15

### Objective

Build 4 frontend finance pages under the `_authenticated/finance/` route group:

1. `/finance/deposits` — Deposit management: list deposits, create deposit (HELD status), change status (HELD → APPLIED/REFUNDED/FORFEITED), delete deposit. Stat cards for Total Held (฿), count by status.
2. `/finance/ar-aging` — AR Aging report: fetch all open invoices, classify into aging buckets (Current, 1-30, 31-60, 61-90, 90+), display summary cards and sortable table.
3. `/finance/monthly-billing` — Monthly billing summary: month picker, filter invoices by creation month, show Total Billed, Collected, Outstanding, Invoice Count, plus subtotal/VAT/total breakdown.
4. `/finance/companies` — Finance companies view: reuse `useCompanies` hook filtered by `industry: 'Finance'`, with search, pagination, and edit dialog.

Also create:
- `useDeposits.ts` hook with full CRUD and unit tests
- `useInvoices.ts` hook with full CRUD
- API backend routes: `invoices`, `payments`, `deposits`
- Shared TypeScript types: Invoice, Payment, Deposit domains

### Detail

**Implementation approach:**
- API routes follow existing Elysia patterns with `authPlugin` guard and `t.Object()` validation
- Invoice auto-generates `INV-YYYYMM-XXXX` numbers; payment creation recalculates invoice status
- Deposit status transitions: HELD → APPLIED/REFUNDED/FORFEITED enforced in API
- Frontend pages use `createFileRoute`, TanStack Query hooks, react-hook-form + zod for forms
- AR Aging classifies invoices by `daysOverdue` using `classifyBucket` pure function
