# T-180: Approval Center + Workflows

**Status:** In Progress
**Assignee:** dev

## Objective

Build a configurable Approval Center for the FMS module that allows multi-step approval workflows for PR, PO, and Budget entities. Users can configure workflow steps per entity type, and approvers can act on pending requests from a unified center.

## Detail

### Implementation Approach
- Add `ApprovalWorkflow` and `ApprovalRequest` Prisma models with an `ApprovalStatus` enum
- Implement CRUD for workflows + create/approve/reject/list-pending API endpoints under `/api/fms/approval-workflows` and `/api/fms/approval-requests`
- Add shared types `ApprovalWorkflow`, `ApprovalRequest`, and related shapes to `packages/shared/src/types/index.ts`
- Create two frontend pages: Approval Center dashboard (`approvals/index.tsx`) and Workflow configuration (`approvals/workflows.tsx`)
- Add "Approval Center" and "Workflows" links in the Sidebar under the Facility Management group

### API Endpoints
- `GET    /api/fms/approval-workflows` ‚Äî list workflows
- `POST   /api/fms/approval-workflows` ‚Äî create workflow
- `GET    /api/fms/approval-workflows/:id` ‚Äî get workflow
- `PUT    /api/fms/approval-workflows/:id` ‚Äî update workflow
- `DELETE /api/fms/approval-workflows/:id` ‚Äî delete workflow
- `GET    /api/fms/approval-requests/pending` ‚Äî list pending for current user
- `POST   /api/fms/approval-requests` ‚Äî create request
- `GET    /api/fms/approval-requests/:id` ‚Äî get request detail
- `POST   /api/fms/approval-requests/:id/approve` ‚Äî approve step
- `POST   /api/fms/approval-requests/:id/reject` ‚Äî reject with reason
