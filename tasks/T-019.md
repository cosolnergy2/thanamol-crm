**Status:** Done
**Assignee:** tester

### Objective

Write Playwright E2E tests covering the project and unit management pages built in T-016, T-017, and T-018.

Test scenarios to cover:

Projects:
- View project list — verify cards render with name, status badge, unit count
- Click project card — verify navigation to ProjectDetail page
- ProjectDetail — verify tabs are visible and Overview tab renders stats
- ProjectDetail Units tab — verify unit list renders for the project
- ProjectDashboard — verify stats cards and at least one chart render without error

Units:
- View UnitList — verify table columns, filter by project, filter by status
- UnitsByProject — verify units are grouped by project
- UnitAvailability — verify grid renders with colored cells per unit status

Error and edge cases:
- Navigate to a non-existent project ID — verify 404 or error state shown
- UnitAvailability with no units for a project — verify empty state message

Tests must use the seeded demo data from T-003. Tests should be independent and not rely on execution order.

Acceptance criteria:
- All test scenarios pass in CI (headed and headless)
- E2E test file(s) placed under `apps/web/e2e/`
- Tests use accessible roles/labels for locators, not brittle CSS selectors
- Screenshots on failure captured automatically
