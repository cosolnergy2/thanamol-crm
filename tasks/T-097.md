# T-097: Cleaning Checklist — add missing fields and Areas & Tasks

**Status:** In Progress
**Assignee:** dev
**Created:** 2026-04-07
**Updated:** 2026-04-07

### Objective

The Cleaning Checklist create form is missing several fields and the entire "Areas & Tasks" section that exists in the Base44 reference. This requires a full-stack enhancement.

Missing scalar fields to add to `CleaningChecklist` Prisma model:
- `checklist_number` — auto-generated string (e.g. "CC-2026-001"), unique
- `site_id` — relation or text reference to site
- `shift` — select with options: Morning, Afternoon, Evening, Night
- `cleaner_id` — reference to user or vendor (dropdown)
- `supervisor_id` — reference to user (dropdown)

Missing JSON field to add:
- `cleaning_areas` — JSON array. Each entry represents one area with:
  - `area_name` (string)
  - `tasks` array — each task has: `task_name` (string), `completed` (boolean), `quality_score` (number, e.g. 1–5 or 0–100)

Acceptance criteria:
- All scalar fields added to `CleaningChecklist` Prisma model with appropriate types and defaults
- `cleaning_areas Json @default("[]")` added to model
- `checklist_number` is auto-generated on creation (e.g. sequential prefix + year + padded sequence)
- A Prisma migration is created using `prisma migrate dev` (not `db push`)
- API create and update routes accept and persist all new fields
- Shared `CleaningChecklist` type and `CreateCleaningChecklistRequest` updated
- Cleaning Checklist create/edit form includes:
  - All new scalar fields (shift dropdown, cleaner dropdown, supervisor dropdown, site dropdown)
  - `checklist_number` displayed as read-only (auto-assigned)
  - "Areas & Tasks" section: add/remove areas, each area has a name input and a list of tasks (add/remove tasks, checkbox for completed, quality score input)
- `pnpm build` passes with zero errors
- `pnpm test` passes

Files to change:
- `apps/api/prisma/schema.prisma`
- Cleaning route (under `apps/api/src/routes/fms/cleaning.ts` or equivalent)
- `packages/shared/src/types/index.ts`
- `apps/web/src/routes/_authenticated/facility-management/cleaning/index.tsx`

### Detail

**Implementation approach:**
- Added new fields to `CleaningChecklist` Prisma model: `checklist_number` (unique), `site_id`, `shift`, `cleaning_areas` (Json), `cleaner_id`, `supervisor_id`
- Created manual migration SQL at `apps/api/prisma/migrations/20260407010000_add_cleaning_checklist_fields/migration.sql` and applied via `prisma migrate deploy`
- Added `CLEANING_SHIFTS` constant and `CleaningShift` type to `packages/shared/src/constants/fms.ts`
- Updated `CleaningChecklist`, `CreateCleaningChecklistRequest`, and `UpdateCleaningChecklistRequest` types in shared package with new fields and `CleaningArea`/`CleaningAreaTask` types
- Auto-generates `checklist_number` in format `CC-YYYY-NNNNN` on POST using sequential count
- Updated API route to accept/persist all new fields; updated Elysia validation schemas accordingly
- Updated frontend form with: Site input, Shift dropdown, Cleaner/Supervisor dropdowns (from users), read-only checklist number, and full "Areas & Tasks" section with add/remove area, add/remove task, completed checkbox, quality score dropdown (1–5)
