# T-157: Asset API + Create form enhancement

**Status:** In Progress
**Assignee:** dev
**Created:** 2026-04-01
**Updated:** 2026-04-01

### Objective

Enhance the Asset create flow to capture all fields introduced in the Phase 1 schema migration (T-152). The current create form is missing dropdown fields (scope type, category, criticality, lifecycle status, condition score) and technical detail fields (brand, model, serial, supplier, purchase price/date, install date, warranty end, specifications, notes, image).

Acceptance criteria:
- API create/update schemas accept all new fields with correct Box (`t.Object()`) validation
- Create form renders all new dropdowns and inputs using constants from `packages/shared`
- Submitting the form persists all fields and the asset appears in the list

### Detail

**Implementation approach:**
- Extend `createAssetSchema` and `updateAssetSchema` in `apps/api/src/routes/fms/assets.ts` with 7 new optional fields using `t.Optional(t.String())` / `t.Optional(t.Number())`, and include them in the Prisma create/update data object
- Update `apps/web/src/routes/_authenticated/facility-management/assets/create.tsx` to add form state for all new fields and render them in 4 structured card sections
- Import `ASSET_SCOPE_TYPES`, `ASSET_CRITICALITIES`, `ASSET_CONDITION_SCORES`, `ASSET_LIFECYCLE_STATUSES` from `@thanamol/shared`
- Add supplier dropdown using `useVendors` hook; `CreateAssetRequest` in shared types already includes all new fields so no shared type changes needed

**Files changed:**
- `apps/api/src/routes/fms/assets.ts`
- `apps/web/src/routes/_authenticated/facility-management/assets/create.tsx`

**API endpoints (updated):**
