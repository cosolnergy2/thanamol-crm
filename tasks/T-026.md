**Assignee:** dev

### Objective
Build the unified approval hub and sale form pages.

Pages:
- /approvals ‚Äî PendingApprovals (unified hub showing all pending quotations and contracts awaiting approval, grouped by type)
- /forms/sale-quotation ‚Äî FormSaleQuotation
- /forms/sale-commercial-proposal ‚Äî FormSaleCommercialProposal

Acceptance criteria:
- /approvals aggregates pending items from both /api/quotations and /api/commercial-quotations (status=pending), grouped by type with counts
- Approve/reject actions on /approvals page gated by manage_contracts permission via PermissionGuard
- FormSaleQuotation and FormSaleCommercialProposal replicate base44 form layouts exactly
- All pages use TanStack Router file-based routing
- Data fetched via TanStack Query
- No hardcoded colors ‚Äî use indigo/teal Tailwind design tokens

Reference base44: PendingApprovals.jsx, FormSaleQuotation.jsx, FormSaleCommercialProposal.jsx

### Detail

**Implementation approach:**
- Create `apps/web/src/hooks/useQuotations.ts` and `useCommercialQuotations.ts` for API hooks fetching `/quotations/pending` and `/commercial-quotations/pending`
- Add `PendingQuotation` and `PendingCommercialQuotation` shared types to `packages/shared/src/types/index.ts`
- Build `apps/web/src/routes/_authenticated/approvals.tsx` ‚Äî tabbed hub with Quotations and Commercial Quotations tabs, using `PermissionGuard` with `manage_contracts` for approve/reject actions
- Build `apps/web/src/routes/_authenticated/forms/sale-quotation.tsx` ‚Äî printable sale quotation form (inline line items, VAT calculation, save action)
- Build `apps/web/src/routes/_authenticated/forms/sale-commercial-proposal.tsx` ‚Äî full commercial proposal form (customer info, warehouse location, rental details table, T&C, footer)

**Files to create or modify:**
- `packages/shared/src/types/index.ts` ‚Äî add `PendingQuotationsResponse`, `PendingCommercialQuotationsResponse`
- `apps/web/src/hooks/useQuotations.ts` ‚Äî `usePendingQuotations`, `useApproveQuotation`, `useRejectQuotation`
- `apps/web/src/hooks/useCommercialQuotations.ts` ‚Äî `usePendingCommercialQuotations`, `useApproveCommercialQuotation`, `useRejectCommercialQuotation`
- `apps/web/src/routes/_authenticated/approvals.tsx`
- `apps/web/src/routes/_authenticated/forms/sale-quotation.tsx`
- `apps/web/src/routes/_authenticated/forms/sale-commercial-proposal.tsx`
- Unit tests alongside each hook file

**API endpoints used:**
- `GET /quotations/pending` ‚Üí `{ data: Quotation[] }` (status=SENT)
- `GET /commercial-quotations/pending` ‚Üí `{ data: CommercialQuotation[] }` (status=SENT)
- `POST /quotations/:id/approve` ‚Äî approve a quotation
- `POST /quotations/:id/reject` ‚Äî reject with reason
- `POST /commercial-quotations/:id/approve` ‚Äî approve a commercial quotation
- `POST /commercial-quotations/:id/reject` ‚Äî reject with reason
