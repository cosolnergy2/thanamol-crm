# T-070: Invoice API enhancement — billing fields, item types, discount, overdue logic, customer name in list

**Status:** Ready to Test
**Assignee:** tester
**Created:** 2026-03-19
**Updated:** 2026-03-19

### Objective

Enhance the existing invoice API (`apps/api/src/routes/invoices.ts`) and Prisma schema to close the gaps identified vs. the base44 reference implementation. The current implementation has basic CRUD but is missing several fields and business logic.

#### Scope

**1. Prisma schema additions (`apps/api/prisma/schema.prisma`)**

Add to the `Invoice` model:
- `invoice_date DateTime?` — the date the invoice was formally issued (distinct from `created_at`). Default to today on create.
- `billing_period_start DateTime?` — start of the billing period this invoice covers.
- `billing_period_end DateTime?` — end of the billing period.
- `discount Float @default(0)` — discount amount applied before final total.
- `days_overdue Int @default(0)` — computed field, days past due date (updated by overdue job).

Update the `InvoiceItem` JSON structure to support an optional `item_type` string field (`Rent`, `Common Fee`, `Water`, `Electricity`, `Parking`, `Other`).

Run `pnpm --filter api db:generate` after schema changes.

**2. API route changes (`apps/api/src/routes/invoices.ts`)**

- `GET /api/invoices` (list): Include customer relation so each invoice in the list response carries `customer: { id, name }`. Currently the list endpoint omits relations, forcing the UI to show raw `customer_id`.
- `POST /api/invoices` (create): Accept the new fields (`invoiceDate`, `billingPeriodStart`, `billingPeriodEnd`, `discount`). Auto-set `invoice_date` to today if not provided.
- `PUT /api/invoices/:id` (update): Accept the new fields.
- Add `PATCH /api/invoices/:id/mark-overdue` endpoint: sets `status = 'OVERDUE'` and updates `days_overdue`. Guard: invoice must be in `SENT` or `PARTIAL` status and `due_date` must be in the past.
- Add `PATCH /api/invoices/:id/void` endpoint: sets `status = 'CANCELLED'`. Guard: invoice must not be `PAID`.
- The `recalculateInvoiceStatus` function in `payments.ts` already handles SENT/PARTIAL/PAID transitions — it must not overwrite OVERDUE status when payments still don't cover the total; update it to preserve OVERDUE when `totalPaid < invoice.total`.

**3. Shared types (`packages/shared/src/types/index.ts`)**

Update `InvoiceItem` type to add `item_type?: string`.

Update `Invoice` type to add:
```typescript
invoice_date: string | null
billing_period_start: string | null
billing_period_end: string | null
discount: number
days_overdue: number
```

Update `CreateInvoiceRequest` and `UpdateInvoiceRequest` to include the new fields:
```typescript
invoiceDate?: string
billingPeriodStart?: string
billingPeriodEnd?: string
discount?: number
```

Run `pnpm build` to regenerate dist types.

**4. Unit test updates (`apps/api/src/routes/invoices.test.ts`)**

- Update existing tests to account for the new fields.
- Add a test for `PATCH /api/invoices/:id/mark-overdue`.
- Add a test for `PATCH /api/invoices/:id/void`.
- Verify `GET /` list response includes customer name (not just customer_id).

#### Acceptance Criteria

- `pnpm build` passes with 0 errors.
- `pnpm test` passes: all existing invoice and payment tests still green, new tests pass.
- `GET /api/invoices` response includes `customer.name` for each invoice.
- `POST /api/invoices` with `invoiceDate`, `billingPeriodStart`, `billingPeriodEnd`, `discount` persists correctly.
- `PATCH /api/invoices/:id/mark-overdue` transitions SENT/PARTIAL → OVERDUE; returns 409 if already PAID/CANCELLED or due date is not past.
- `PATCH /api/invoices/:id/void` transitions to CANCELLED; returns 409 if PAID.
- Payment creation does not overwrite OVERDUE back to SENT when balance is not fully covered.

### Detail

**Implementation approach:**
- Add `invoice_date`, `billing_period_start`, `billing_period_end`, `discount`, `days_overdue` fields to the `Invoice` model in `schema.prisma`. `InvoiceItem` keeps using `Json` so `item_type` is handled at the TypeScript/API schema level without a DB enum.
- Enhance `GET /api/invoices` list to `include: { customer: { select: { id, name } } }` so each record carries customer name.
- Add new optional fields to `createInvoiceSchema` and `updateInvoiceSchema` in Elysia Box types (`t.Object`). Auto-default `invoice_date` to today when not provided on create.
- Add `PATCH /:id/mark-overdue` — guards: status must be SENT or PARTIAL and `due_date` must be in the past. Sets `status = OVERDUE` and computes `days_overdue`.
- Add `PATCH /:id/void` — guard: status must not be PAID. Sets `status = CANCELLED`.
- Fix `recalculateInvoiceStatus` in `payments.ts`: when `totalPaid < invoice.total`, preserve OVERDUE status instead of downgrading to SENT.
- Update shared types: `InvoiceItem.item_type?`, new `Invoice` fields, new request fields.
