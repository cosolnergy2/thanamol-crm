# T-168: PR detail/edit + status workflow

**Status:** Ready to Test
**Assignee:** dev
**Created:** 2026-04-01
**Updated:** 2026-04-01

### Objective

Build the PR detail view and edit form, and implement the Submit / Approve / Reject status workflow. Currently the PR detail page does not display the new fields added in T-167 (conditions, documents, item types, Buy/Rent). The status workflow buttons are missing so PRs cannot be moved from Draft through the approval chain.

Acceptance criteria:
- Detail page displays all PR fields: header info (PR number, company, site, unit, purpose, required date, PM schedule link), item lines table with Type and Buy/Rent columns, Conditions section rendered as read-only summary, Documents list
- Edit form mirrors the T-167 create form and pre-fills with existing PR data; only allowed when PR is in Draft status
- Submit button visible on Draft PRs — moves status to Submitted
- Approve button visible on Submitted PRs — moves status to Approved
- Reject button visible on Submitted PRs — prompts for reason, moves status to Rejected
- Status badge on detail page updates immediately after workflow action

### Detail

**Implementation approach:**
- Updated `$prId/index.tsx` to show all PR fields: PR number/status/priority badges in header, General Info card (company/site/unit/purpose/required date/PM schedule link), Items table with Type and Buy/Rent columns, Conditions card (installments table, flags, timeline), Documents list, Notes
- Added `useSubmitPurchaseRequest` hook call to detail page for the DRAFT Submit button
- Added Edit button on DRAFT/REJECTED statuses navigating to `$prId/edit`
- Created `$prId/edit.tsx`: full edit form mirroring create.tsx, pre-filled via `useEffect` from `usePurchaseRequest` data, guards non-DRAFT status with read-only message
- Status guard: edit page returns read-only message with back button when status != DRAFT
- Fixed navigation paths to use no-trailing-slash form matching TanStack Router route tree
- Regenerated `routeTree.gen.ts` via `@tanstack/router-cli generate`
