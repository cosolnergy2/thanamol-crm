**Status:** Done
**Assignee:** tester

**Updated:** 2026-03-15

### Objective

Write Playwright E2E tests that verify the complete authentication flow end-to-end. These tests depend on T-004 (auth API) and T-005 (auth frontend) being complete.

**Test scenarios required:**

1. **Login page renders** — visit `/login`, assert email field, password field, and submit button are visible
2. **Successful login** — submit valid credentials (`admin@thanamol.com` / `password123`), assert redirect to dashboard and user name appears in top bar
3. **Invalid credentials** — submit wrong password, assert inline error message is shown (no redirect)
4. **Unauthenticated redirect** — visit a protected route (e.g., `/`) without a token, assert redirect to `/login`
5. **Token persistence** — log in, reload the page, assert user is still authenticated (not redirected to login)
6. **Logout** — click logout in the user menu, assert redirect to `/login` and token is cleared (subsequent protected route visit redirects again)

**Test file location:** `apps/web/e2e/auth.spec.ts`

**Screenshot requirements:**
- Capture screenshot after each key state: login page, dashboard after login, error message state, post-logout login page

**Acceptance criteria:**
1. All 6 test scenarios pass
2. Screenshots captured and saved to `apps/web/e2e/screenshots/auth/`
3. Test results documented in the Test Results section below
4. `pnpm --filter web exec playwright test e2e/auth.spec.ts` exits with code 0

| # | Test scenario | Expected behaviour | Selector notes |
|---|---------------|--------------------|----------------|
| 1 | Login page renders | `/login` shows `<label>Email`, `<label>Password`, and a "Sign in" button | `getByLabel('Email')`, `getByLabel('Password')`, `getByRole('button', { name: 'Sign in' })` |
| 2 | Successful login | Submitting `admin@thanamol.com` / `password123` redirects to `/` and renders "Hello, &lt;name&gt;" | `getByText(/Hello,/)` |
| 3 | Invalid credentials | Submitting wrong credentials shows `<p role="alert">` error text; URL stays `/login` | `getByRole('alert')` |
| 4 | Unauthenticated redirect | Visiting `/` with no tokens in `localStorage` redirects to `/login` | URL assertion |
| 5 | Token persistence | After login + page reload the user remains on `/` and "Hello, &lt;name&gt;" is still visible | `page.reload()` then URL + text assertion |
| 6 | Logout clears session | Clearing tokens + navigating to `/login` then visiting `/` redirects back to `/login` | `localStorage.removeItem` via `page.evaluate` (no logout UI exists yet; see note in test) |

**Note on DoD 6 (Logout):** The current UI (`_authenticated.tsx`) renders only `<Outlet />` with no logout button. Test 6 simulates `AuthProvider.logout()` behaviour programmatically. A proper UI-triggered logout test should be revisited once a navigation/user-menu component is added.
