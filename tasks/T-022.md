### Objective
Write Playwright E2E tests covering lead management and deal pipeline workflows built in T-020 and T-021.

Scope:
- Lead list: load, search, filter by status/source/assignedTo, pagination
- Lead detail: view full info, activity timeline visible
- Lead CRUD: create, edit, delete a lead
- Deal pipeline: Kanban board renders with stage columns and deal cards
- Deal stage transition: drag a deal card from one column to another, verify persistence
- Deal CRUD: create, edit, delete a deal
- Deal detail: view deal with customer/lead relations

Acceptance criteria:
- Tests run via `pnpm --filter web exec playwright test`
- All tests pass against the local dev stack
- Test files placed under apps/web/e2e/
- Covers happy paths and key error states (e.g., create with missing required fields)

### Definition of Done

Test file: `apps/web/e2e/T-022-lead-deal-pipeline.spec.ts`

**Lead tests**
1. Lead list page renders with stats cards (New, Contacted, Qualified, Converted) and "Lead Inbox" heading
2. Create a new lead via the "Add Lead" dialog — lead appears in the table
3. Create a lead without a title shows validation error (toast)
4. Filter leads by status using the status dropdown — table updates accordingly
5. Navigate to lead detail page — form fields are populated
6. Edit a lead field and save — success toast is shown
7. Convert a lead to deal from the list page — navigates to /deals after confirmation

**Deal tests**
8. Deal pipeline page renders with stage columns: Prospecting, Qualification, Proposal, Negotiation, Won, Lost
9. Create a new deal via the "New Deal" dialog — deal card appears in the Prospecting column
10. Create a deal without a title shows validation error (toast)
11. Navigate to deal detail page — Deal Detail heading and stage badge visible
12. Edit a deal field and save — success toast is shown
13. Move a deal to a different stage via the "Move to..." select — confirmation dialog appears and on confirm deal moves
