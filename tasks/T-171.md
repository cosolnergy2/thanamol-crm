# T-171: Vendor API + registration form overhaul

**Status:** Ready to Dev
**Assignee:** dev
**Created:** 2026-04-01
**Updated:** 2026-04-01

### Objective
Overhaul the vendor registration form and API to capture the full set of vendor data required by the FMS module. The existing vendor create form is incomplete — it is missing critical identification fields (Vendor Code, Company Registration, Tax ID), extended contact information (additional contacts list), procurement configuration (Service Tags, Payment Terms, Credit Limit), and Default Conditions (Payment Installment, WHT, VAT, Retention, Warranty, Credit Terms, Timeline, Late Penalty, Insurance).

Acceptance criteria:
- Vendor Code auto-generates on the create form (read-only display)
- Type dropdown is populated (Supplier, Contractor, Consultant, Service Provider, etc.)
- Company Registration, Legal Name, Display Name, Tax ID fields exist and save correctly
- Address, Phone, Email, Primary Contact fields exist
- Additional Contacts section allows adding multiple rows (name, phone, email, position) dynamically
- Service Tags is a multi-select field covering relevant service categories
- Supplier Type, Rating (1-5 stars), Status dropdowns exist
- Payment Terms and Credit Limit fields exist
- Default Conditions section mirrors the Conditions section from PR/PO forms (Payment Installment, WHT, VAT, Retention, Warranty, Credit Terms, Timeline, Late Penalty, Insurance)
- Submitting the form creates the vendor and it appears in the vendor list
