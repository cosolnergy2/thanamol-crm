# T-012: E2E tests for RBAC and language switching

**Status:** Done
**Assignee:** tester
**Created:** 2026-03-15
**Updated:** 2026-03-15

### Objective

Write Playwright E2E tests that verify permission-based UI visibility and the language toggle. This task depends on T-009 (RBAC API), T-010 (frontend guards), and T-011 (translations) being complete.

**Test file location:** `apps/web/e2e/T-012-rbac-i18n.spec.ts`

**Test scenarios required:**

1. **Admin sees full nav** — log in as an admin user, assert all major nav sections are visible in the sidebar
2. **Limited user sees restricted nav** — log in as a user with a limited role, assert that nav sections outside their permissions are not rendered
3. **PermissionGuard hides content** — visit a page that has a `PermissionGuard`-wrapped section; assert the restricted section is not visible to the limited user
4. **Language toggle — EN to TH** — click the EN/TH toggle in the TopBar, assert that at least three UI strings change to Thai
5. **Language toggle — TH to EN** — click toggle again, assert strings revert to English
6. **Language persists on reload** — switch to TH, reload the page, assert UI still shows Thai strings

**Screenshot requirements:**
- Admin nav (all items visible)
- Limited user nav (restricted items hidden)
- Page with PermissionGuard content hidden
- Language switch to TH
- Language switch back to EN
- Language persisted after reload

**Acceptance criteria:**
1. All 6 test scenarios pass with the full stack running
2. Screenshots saved to `apps/web/e2e/screenshots/T-012/`
3. Test results documented in the Test Results section below
4. `pnpm --filter web exec playwright test apps/web/e2e/T-012-rbac-i18n.spec.ts` exits with code 0

### Definition of Done

**Test file:** `apps/web/e2e/T-012-rbac-i18n.spec.ts`

**DoD 1 — Admin sees full nav:** Log in as `admin@thanamol.com`. Assert that all major nav section labels are visible in the sidebar: Dashboard, My Dashboard, Projects, Customers, Leads & Deals, Units/Products, Quotations, Contracts, Finance, Utilities, Service, Documents, Meeting Minutes, Form List, Reports, and Settings.

**DoD 2 — Regular user sees restricted nav:** Log in as `user1@thanamol.com` (role: `User`, has `view_reports: true` only). Assert that nav items requiring permissions the user does not have (Finance, Settings, Contracts, etc.) are NOT rendered in the sidebar.

**DoD 3 — PermissionGuard hides content:** Log in as `user1@thanamol.com`. Navigate to a page that requires elevated permissions. Assert that the restricted section/content is not visible. (Uses the roles/settings page which requires `manage_roles` permission.)

**DoD 4 — Language toggle EN to TH:** After logging in as admin, click the EN/TH toggle button in the TopBar. Assert that at least three sidebar navigation labels change to their Thai translations (e.g., "โครงการ", "ลูกค้า", "ใบเสนอราคา").

**DoD 5 — Language toggle TH to EN:** Click the toggle a second time. Assert that the labels revert to English (e.g., "Projects", "Customers", "Quotations").

**DoD 6 — Language persists on reload:** Switch to Thai, reload the page, assert the UI still shows Thai strings. (Language preference is stored in `localStorage` under key `preferredLanguage`.)
