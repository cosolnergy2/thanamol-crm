### Objective

Port the main application layout shell from base44 into the `_authenticated` layout route. This is the persistent chrome (sidebar + top bar) that wraps every authenticated page.

**Source:** `/Users/jettapat/personal/7tapat/projects/propertyflow-crm/src/Layout.jsx`
**Target:** `apps/web/src/routes/_authenticated.tsx` (layout route)

**Sidebar requirements:**
- Collapsible (icon-only vs full width)
- 15 navigation sections, each with sub-items:
  1. Dashboard
  2. Projects
  3. Customers
  4. Leads & Deals
  5. Units
  6. Quotations
  7. Contracts
  8. Finance
  9. Utilities
  10. Service
  11. Documents
  12. Meeting Minutes
  13. Forms
  14. Reports
  15. Settings
- Active route highlighted using TanStack Router's `useRouterState` or link `activeProps`
- RBAC: hide nav sections the current user's role does not have access to

**Top bar requirements:**
- User profile dropdown (show name, logout option)
- Notification bell icon (badge count, no functionality needed in this task)
- Language switcher toggle (EN / TH) — store preference in localStorage

**Mobile:**
- Hamburger menu that opens sidebar as an overlay/drawer on screens < lg

**Adaptation notes:**
- Replace `Link`, `useNavigate`, `useLocation` (React Router) with TanStack Router equivalents (`<Link>`, `useNavigate`, `useRouterState`)
- Replace `createPageUrl("PageName")` calls with literal TanStack Router path strings
- Use UI components from T-006 (depends on T-006 being complete)

**Acceptance criteria:**
1. Sidebar renders all 15 sections with correct icons and sub-items
2. Sidebar collapses to icon-only mode and expands back
3. Currently active route is visually highlighted in the sidebar
4. Top bar shows authenticated user's name and a working logout button
5. Language toggle switches between EN and TH labels (no i18n content needed, just state)
6. Mobile hamburger opens/closes the sidebar drawer
