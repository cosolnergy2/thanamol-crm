### Objective

Build the unit/property pages, porting UI from base44 source pages.

Pages to create:
- `/units` — UnitList with data table, filter by project, status, type, floor; pagination
- `/units/by-project` — UnitsByProject with units grouped and displayed per project
- `/units/availability` — UnitAvailability matrix/grid view showing unit status per floor per project

Technical requirements:
- Use TanStack Router file-based routes under `apps/web/src/routes/units/`
- Use TanStack Query for data fetching from T-016 endpoints
- UnitAvailability page renders a grid: rows = floors, columns = units per floor, cells colored by status
- Use Badge/color tokens from design system for unit status (AVAILABLE, RESERVED, SOLD, BOOKED)
- Clicking a unit cell or row opens a detail dialog or navigates to unit detail
- Loading states use skeletons; error states show actionable messages
- Responsive: table/grid must be horizontally scrollable on smaller screens

Acceptance criteria:
- UnitList table renders with correct columns (unit number, project, type, floor, status, price)
- Filters (project, status, type, floor) update results and reset to page 1
- UnitsByProject groups units under their project with a collapsible or sectioned layout
- UnitAvailability grid renders correctly with status-coded cells
- No hardcoded colors or arbitrary Tailwind values

### Detail

**Implementation approach:**
- Create `useUnits` hook with TanStack Query for unit CRUD and availability data fetching from T-016 API endpoints
- Build `UnitList` page as a card grid with search + status/type filters, "Add Unit" dialog using shared `Unit` types
- Build `UnitsByProject` page with Accordion component grouping units under each project with per-project stats
- Build `UnitAvailability` page showing available units in a table view (matching base44: filter by building/type/size, CSV export button)
- Status color tokens use design system classes (emerald=available, amber=reserved, blue=occupied, slate=maintenance)

**API endpoints used (T-016):**
- `GET /api/units` — paginated list with query params: projectId, status, type, floor, search, minArea, maxArea, minPrice, maxPrice, page, limit
- `GET /api/units/:id` — single unit
- `GET /api/units/availability` — grouped by project with UnitAvailabilityByProject[]
- `POST /api/units` — create unit
- `PUT /api/units/:id` — update unit
- `DELETE /api/units/:id` — delete/soft-delete unit
