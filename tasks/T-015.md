### Objective

Write Playwright E2E tests covering the CRM entity pages built in T-013 and T-014.

Test scenarios to cover:

Customer:
- Create a new customer (INDIVIDUAL type) — fill form, submit, verify appears in list
- Create a new customer (CORPORATE type) — fill form, submit, verify appears in list
- View customer list — verify table columns, pagination controls visible
- Search customers by name — verify results filter correctly
- Filter customers by status (ACTIVE, PROSPECT) — verify results update
- Filter customers by type — verify results update

Contact:
- Create a new contact linked to an existing customer
- View contact list — verify table and filter by customer dropdown

Company:
- View company list — verify table, search by name, filter by industry

Error cases:
- Submit CustomerCreate form with missing required fields — verify inline validation messages
- Search with no results — verify empty state message shown

Tests must use the test user seeded by T-003. Tests should be independent and not rely on execution order.

Acceptance criteria:
- All test scenarios pass in CI (headed and headless)
- E2E test file(s) placed under `apps/web/e2e/`
- Tests reference page objects or locators by accessible roles/labels, not brittle CSS selectors
- Screenshots on failure captured automatically
