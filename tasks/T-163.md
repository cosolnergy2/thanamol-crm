# T-163: Inventory API + create form enhancement

**Status:** In Progress
**Assignee:** dev
**Created:** 2026-04-01
**Updated:** 2026-04-01

### Objective

Enhance the Inventory Item API and create/detail forms to expose all new schema fields added in T-153. Users currently cannot set Item Type, Barcode, Company/Project/Site scope, Vendor & Lead Time, Specifications, or image attachments when creating an inventory item. Stock level fields (reorder point, min/max) must also be editable.

Acceptance criteria:
- API create and update schemas accept `item_type`, `barcode`, `company_id`, `site_id`, `specifications` (Json), `vendor_id`, `lead_time_days`, `photos` (Json)
- Item Code is auto-generated and displayed read-only on the create form
- Type dropdown sourced from `INVENTORY_ITEM_TYPES` constant (Spare Part, etc.)
- Category dropdown sourced from existing constants (HVAC, Electrical, Plumbing, etc.)
- Unit dropdown sourced from constants (Piece, kg, liter, etc.)
- Company / Project / Site cascading selectors
- Vendor & Lead Time section: vendor selection + lead_time_days numeric input
- Specifications textarea (Json stored)
- Stock level fields: current quantity, reorder point, min quantity, max quantity, unit price
- Image upload stored in photos Json field
- All dropdown values sourced from constants in `packages/shared` — no hardcoded strings

### Detail

### Implementation Approach
- Update API create/update Elysia schemas to include all new fields (itemType, barcode, companyId, siteId, specifications, vendorId, leadTimeDays, photos, vendorItemCode, backupVendorId, backupVendorItemCode) and map them to Prisma
- Restructure the create form into 5 sections: Basic Info, Ownership & Storage, Stock & Pricing, Vendor & Lead Time, Additional
- Update detail page to display all new fields in organized sections
- Create an edit page that mirrors the create form with pre-filled item data
