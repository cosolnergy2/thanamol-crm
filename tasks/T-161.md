# T-161: Calibration Record API + dedicated create page

**Status:** In Progress
**Assignee:** dev
**Created:** 2026-04-01
**Updated:** 2026-04-01

### Objective

Replace the current inline dialog for creating calibration records with a dedicated create page, and update the API to support all new fields added in T-152. The new page must include a dynamic Calibration Results table where each row captures a measurement parameter and auto-calculates deviation and pass/fail status.

Acceptance criteria:
- API create and update schemas accept `calibration_number` (auto-gen), `frequency_days`, `calibration_type`, `calibration_standard`, `certificate_number`, `cost`, `results` (Json)
- Dedicated create page at `calibrations/create.tsx` (removes inline dialog)
- Calibration Number is auto-generated and read-only
- Next Calibration Date auto-calculates from calibration date + frequency_days
- Calibration Results table: dynamic rows with Parameter, Standard, Measured, Deviation (auto-calc = Measured − Standard), Tolerance, Status (Pass/Fail based on tolerance)
- Calibration Type dropdown: Internal, External, Factory
- All dropdown values sourced from shared constants

### Detail

**Implementation approach:**
- Extend API create/update schemas with all new CalibrationRecord fields; add auto-generation of `calibration_number` (CAL-NNNNN pattern) when not provided
- Create dedicated create page with two sections: Basic Information and Calibration Results dynamic table
- Deviation = measured - standard (read-only); Status = Pass if |deviation| <= tolerance else Fail (read-only badge)
- Next Calibration Date auto-calculates from calibration date + frequency_days
- Update list page to replace inline dialog button with a link to the new create page
- Detail and edit pages follow the `$calibrationId/` nested route pattern (T-162)
