# T-167: PR API + form overhaul

**Status:** Done
**Assignee:** dev
**Created:** 2026-04-01
**Updated:** 2026-04-01

### Objective

Overhaul the Purchase Request API and create form to expose all new schema fields added in T-154. Currently the PR form is missing Company/Site/Unit scope, Required Date, Purpose, PM Schedule link, per-line item configuration (Type, Buy/Rent), Budget Code, Supplier, and the full Conditions section (Payment Installment, WHT, VAT, Retention, Warranty, Credit Terms, Timeline, Late Penalty, Insurance). Documents upload is also missing.

Acceptance criteria:
- API create and update schemas accept `company_id`, `site_id`, `unit_id`, `required_date`, `purpose`, `pm_schedule_id`, `documents` (Json), `conditions` (Json)
- Company / Site / Unit cascading selectors on the form
- Required Date date-picker field
- Purpose dropdown sourced from `PR_PURPOSES` constant (Stock Replenishment / Work Order / Project / Emergency / Service Contract / Other)
- Optional PM Schedule link selector
- Item lines include Type dropdown sourced from `PR_ITEM_TYPES` constant (Material / Labor / Contract / Engineering / Service / Asset / Inventory / PM / Other) and Buy/Rent toggle
- Budget Code input per item line
- Conditions section: Payment Installment rows (milestone + amount + due date), WHT toggle + rate, VAT toggle + rate, Retention toggle + %, Warranty text, Credit Terms text, Timeline text, Late Penalty text, Insurance toggle + details
- Documents upload (stored as Json array of URLs/names)
- All dropdown values sourced from constants in `packages/shared` — no hardcoded strings
