# T-084: Contact form — add LINE ID and Decision Maker fields

**Status:** In Progress
**Assignee:** dev
**Created:** 2026-03-24
**Updated:** 2026-03-24

### Objective

Add 2 missing fields to the Contact model and form compared to the Base44 reference:
1. LINE ID — text input field
2. Is Decision Maker — checkbox (separate from existing "Is Primary Contact")

Acceptance criteria:
- LINE ID text input appears in Add New Contact dialog (after phone/email fields)
- "Decision Maker" checkbox appears separate from "Primary Contact" checkbox
- Both fields are stored and retrieved via the API
- "Decision Maker" badge is shown in the Flags column of the contacts table

### Detail

Implementation approach:
- Add `line_id String?` and `is_decision_maker Boolean @default(false)` to Prisma Contact model
- Add `lineId` and `isDecisionMaker` to shared Contact type and CreateContactRequest type
- Add `lineId` and `isDecisionMaker` to Elysia create/update schemas; map to snake_case in Prisma calls
- Add LINE ID input and Decision Maker checkbox to the frontend Add Contact dialog
- Show Decision Maker badge (teal) in the Flags column alongside the existing Primary badge

Files modified:
- `apps/api/prisma/schema.prisma` — added `line_id` and `is_decision_maker` to Contact model
- `apps/api/prisma/migrations/20260324150457_add_contact_line_id_decision_maker/migration.sql` — new migration
- `packages/shared/src/types/index.ts` — added fields to Contact and CreateContactRequest types
- `apps/api/src/routes/contacts.ts` — added to create/update schemas and Prisma calls
- `apps/web/src/routes/_authenticated/contacts.tsx` — added LINE ID input, Decision Maker checkbox, Decision Maker badge
- `apps/api/src/routes/contacts.test.ts` — added tests for new fields

API changes:
- POST /api/contacts — body now accepts `lineId?: string` and `isDecisionMaker?: boolean`
- PUT /api/contacts/:id — body now accepts `lineId?: string` and `isDecisionMaker?: boolean`

Documentation referenced:
