# T-028 — API — Contract CRUD + Multi-Level Approval + Lease Agreement CRUD

**Status:** In Progress
**Assignee:** dev

## Objective

Implement two API route files: Contract and LeaseAgreement management.

### Contract API
- `GET /api/contracts` — list with pagination and filters (status, type, customerId, projectId)
- `GET /api/contracts/:id` — single contract with full relations
- `POST /api/contracts` — create (auto-generate contract number CT-YYYYMM-XXXX)
- `PUT /api/contracts/:id` — update
- `DELETE /api/contracts/:id` — delete (only DRAFT status)
- `GET /api/contracts/pending` — contracts with status=PENDING_APPROVAL
- `GET /api/contracts/expiring` — contracts expiring within N days (query param `days`, default 30)
- `POST /api/contracts/:id/approve` — approve (requires manage_contracts permission)
- `POST /api/contracts/:id/reject` — reject with reason (requires manage_contracts permission)

### Lease Agreement API
- `GET /api/lease-agreements` — list (filter by contractId)
- `GET /api/lease-agreements/:id` — single with contract relation
- `POST /api/lease-agreements` — create linked to contract
- `PUT /api/lease-agreements/:id` — update
- `DELETE /api/lease-agreements/:id` — delete

## Acceptance Criteria

1. Contract number auto-generated as CT-YYYYMM-XXXX (padded 4-digit sequence)
2. DELETE contract returns 409 if status is not DRAFT
3. Approval transitions: PENDING_APPROVAL → APPROVED (single level, sets approved_by)
4. Reject sets status to CANCELLED and stores reason in terms field
5. Approve/Reject require manage_contracts permission (403 if missing)
6. Expiring endpoint filters by end_date within N days from now
7. All list endpoints return paginated responses
8. LeaseAgreement linked to Contract via contract_id

## Detail

### Implementation Approach
- Follow quotations.ts pattern exactly: authPlugin, guard, Box types (t.Object)
- Contract number generation: CT-YYYYMM-XXXX, count existing with same prefix
- Multi-level approval: PENDING_APPROVAL → APPROVED → ACTIVE (two steps)
- Expiring endpoint: where end_date <= now + N days AND status = ACTIVE
- Only DRAFT contracts can be deleted (409 otherwise)
- Shared types in packages/shared/src/types/index.ts
