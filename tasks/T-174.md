# T-174: Vendor Invoice create form

**Status:** In Progress
**Assignee:** dev
**Created:** 2026-04-01
**Updated:** 2026-04-01

### Objective
Build a dedicated create form for vendor invoices. Currently there is no frontend create page for vendor invoices. The form needs to capture invoice metadata, link to a PO for auto-population, and support PDF attachment upload. A submission history section should display on the invoice detail page.

Acceptance criteria:
- Vendor selection dropdown populates the form context
- PO Reference selection — selecting a PO auto-populates vendor and amount fields
- Invoice Number field (manual entry)
- Invoice Amount field
- PDF upload field for attaching the invoice document
- Submitting the form creates the invoice record and it appears in the vendor invoices list
- Invoice detail page shows a Submission History section listing each time the invoice was submitted/resubmitted with timestamps

### Detail
**Implementation approach:**
- Add `pdfUrl` and `submissionHistory` fields to `createInvoiceSchema` and `updateInvoiceSchema` in the API, persisting them to `pdf_url` and `submission_history` columns (already in Prisma schema).
- Add `POST /:id/submit` endpoint to append a submission event (timestamp + notes) to `submission_history` JSON array.
- Create `vendors/invoices/create.tsx` — full-page form: vendor dropdown, PO reference dropdown (auto-populates vendor + total amount), invoice number, amount, PDF URL input, submit button.
- Create `vendors/invoices/$invoiceId.tsx` — detail page showing invoice metadata and a Submission History section.
- Update `vendors/invoices/index.tsx` list page: link invoice number to detail page, redirect Create button to the new `/create` route instead of the dialog.

**Files to create/modify:**
- `apps/api/src/routes/fms/vendor-invoices.ts` — add pdfUrl/submissionHistory to schemas, add submit endpoint
- `apps/web/src/routes/_authenticated/facility-management/vendors/invoices/create.tsx` — new create form page
- `apps/web/src/routes/_authenticated/facility-management/vendors/invoices/$invoiceId.tsx` — new detail page
- `apps/web/src/routes/_authenticated/facility-management/vendors/invoices/index.tsx` — update list page
- `apps/web/src/hooks/useVendorInvoices.ts` — add useSubmitVendorInvoice hook

**API endpoints added:**
- `POST /api/fms/vendor-invoices/:id/submit` — body: `{ notes?: string }`, appends `{ submittedAt, notes }` to submission_history
