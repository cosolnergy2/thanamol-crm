### Objective

The Fire Equipment create/edit form is missing 7 fields that exist in the Base44 reference. These require a full-stack addition: Prisma model column additions, a new migration, API schema and handler updates, shared type updates, and UI form fields.

Missing fields to add:
- `manufacturer` — text input
- `model` — text input
- `serial_number` — text input
- `capacity_size` — text input (e.g. "5kg", "9L")
- `installation_date` — date picker
- `condition` — select with options: Good, Fair, Poor, Requires Replacement
- `certification_number` — text input

Acceptance criteria:
- All 7 fields added to `FireEquipment` Prisma model with appropriate types and `@default` values where needed
- A Prisma migration is created using `prisma migrate dev` (not `db push`)
- API create and update routes accept and persist all 7 fields
- Shared `FireEquipment` type and `CreateFireEquipmentRequest` type include all 7 fields
- Fire Equipment create form displays all 7 fields in a logical grouping
- Fire Equipment edit form populates all 7 fields from saved data
- `pnpm build` passes with zero errors
- `pnpm test` passes

Files to change:
- `apps/api/prisma/schema.prisma`
- API fire-equipment route (under `apps/api/src/routes/fms/compliance.ts` or dedicated route file)
- `packages/shared/src/types/index.ts`
- `apps/web/src/routes/_authenticated/facility-management/compliance/fire-equipment.tsx`

### Detail

Implementation approach:
- Add 7 new nullable columns to `FireEquipment` Prisma model: manufacturer, model, serial_number, capacity_size, installation_date (DateTime?), condition, certification_number
- Run `prisma migrate dev` to create migration
- Extend `FireEquipment` shared type and `CreateFireEquipmentRequest` with new fields; add `FIRE_EQUIPMENT_CONDITIONS` constant
- Extend `createFireEquipmentSchema` / `updateFireEquipmentSchema` and Prisma create/update calls in the API route
- Add form fields to the frontend dialog; condition as Select with options Good/Fair/Poor/Requires Replacement; installation_date as date input
