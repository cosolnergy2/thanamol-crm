# T-091: Add Stock/Store toggle to asset creation form

**Status:** In Progress
**Assignee:** dev
**Created:** 2026-04-07
**Updated:** 2026-04-07

### Objective

The asset creation form is missing a "เป็น Stock/Store (จัดการแยกโดยเฉพาะ)" boolean toggle identified in the bug report. This field allows staff to flag an asset as a stock/store-managed item so it can be handled separately in the system.

Fix requires a full-stack change: add `is_stock_store` boolean field to the Asset Prisma model, create a migration, expose the field through the API, and add the toggle to both the asset create and edit forms.

Acceptance criteria:
- `is_stock_store Boolean @default(false)` column exists on the `Asset` model in Prisma schema
- A Prisma migration is created and applied (using `prisma migrate dev`, not `db push`)
- Asset API create and update routes accept and persist `is_stock_store`
- Asset create form displays the "เป็น Stock/Store (จัดการแยกโดยเฉพาะ)" toggle, defaulting to off
- Asset edit form displays the toggle and correctly reflects the saved value
- Value round-trips correctly: toggling on, saving, and reopening the edit form shows the toggle on
- `pnpm build` passes with zero errors
- `pnpm test` passes

### Detail

Implementation approach:
- Added `is_stock_store Boolean @default(false)` to the Asset model in `apps/api/prisma/schema.prisma`
- Created migration `20260407000000_add_asset_is_stock_store` with `ALTER TABLE "Asset" ADD COLUMN "is_stock_store" BOOLEAN NOT NULL DEFAULT false`
- Added `isStockStore: t.Optional(t.Boolean())` to both `createAssetSchema` and `updateAssetSchema` in `apps/api/src/routes/fms/assets.ts`
- Added `is_stock_store` mapping in the POST create handler (`body.isStockStore ?? false`) and PUT update handler (`body.isStockStore`)
- Added `is_stock_store?: boolean` to `Asset` type and `isStockStore?: boolean` to `CreateAssetRequest` in `packages/shared/src/types/index.ts`
- Added `isStockStore` state variable and toggle UI to the Ownership & Location card in both create and edit forms, placed after the Location Detail field
- Edit form populates `isStockStore` from `asset.is_stock_store ?? false` in `useEffect`

Files changed:
