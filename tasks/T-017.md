### Objective

Build the project management pages, porting UI from base44 source pages.

Pages to create:
- `/projects` — ProjectList with card/grid view, search, status filters, pagination
- `/projects/$projectId` — ProjectDetail with tab navigation: Overview, Units, Contracts (and other tabs present in base44)
- `/projects/dashboard` — ProjectDashboard with summary stats cards and charts

Technical requirements:
- Use TanStack Router file-based routes; ProjectDetail uses dynamic segment `$projectId`
- Use TanStack Query for all data fetching; prefetch project detail on hover if feasible
- Use UI components from T-006 for cards, tables, badges, tabs
- ProjectDashboard charts: use Recharts or the charting library already used in the project
- Loading states use skeletons; error states show actionable messages
- Design system: indigo/teal palette, no arbitrary colors

Acceptance criteria:
- ProjectList renders as a card grid with project name, status badge, unit count
- Clicking a project card navigates to ProjectDetail
- ProjectDetail tabs render correct data per tab (units list, overview stats, etc.)
- ProjectDashboard shows unit status distribution and at least one revenue/activity chart
- Responsive layout on mobile
- No hardcoded values
