### Objective

Build the authentication frontend layer: login page, AuthContext provider, and protected route wrapper. This unblocks all subsequent frontend tasks that require an authenticated user.

**Deliverables:**

1. **Login page** at `/login` route
   - Email and password fields
   - Submit button with loading state
   - Inline error message on failed login (improvement over base44 which used an alert)
   - Redirect to `/` (dashboard) on success

2. **AuthContext** (`apps/web/src/context/AuthContext.tsx`)
   - `currentUser` state (null when logged out)
   - `login(email, password)` — calls POST /api/auth/login, stores tokens in localStorage
   - `logout()` — calls POST /api/auth/logout, clears localStorage, redirects to /login
   - Auto-refresh: schedule a token refresh 30 seconds before access token expires
   - Export `useAuth()` hook

3. **API client** (`apps/web/src/lib/api-client.ts`)
   - Fetch wrapper that auto-attaches `Authorization: Bearer <token>` header
   - Handles 401 responses by attempting a token refresh then retrying once
   - Base URL from env var `VITE_API_URL`

4. **Protected route layout** (`apps/web/src/routes/_authenticated.tsx`)
   - Redirects to `/login` if no valid token present
   - Wraps all authenticated pages

**Reference:** `/Users/jettapat/personal/7tapat/projects/propertyflow-crm/src/lib/AuthContext.jsx`

**Acceptance criteria:**
1. Login page renders with email/password fields and submit button
2. Successful login stores token and redirects to dashboard
3. Failed login (wrong password) shows an inline error message
4. Navigating to any `/_authenticated` route without a token redirects to /login
5. Token auto-refresh fires before expiry and keeps session alive
6. Logout clears token and redirects to /login
