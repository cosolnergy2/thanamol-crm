# T-165: Petty Cash form enhancement

**Status:** Ready to Dev
**Assignee:** dev
**Created:** 2026-04-01
**Updated:** 2026-04-01

### Objective

Enhance the Petty Cash Transaction create form to expose the new schema fields added in T-153. Users currently cannot associate a transaction with a specific site, budget code, or responsible person. These three fields must be added to both the API and the frontend form.

Acceptance criteria:
- API create and update schemas accept `site_id`, `budget_code`, `responsible_person_id`
- Create form shows Site selector (FK to Site), Budget Code text input, Responsible Person dropdown (FK to User)
- Transaction list displays site name and responsible person name columns
- Detail page shows all three new fields
- No hardcoded values — site list sourced from API, user list sourced from API
