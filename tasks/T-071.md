# T-071: Invoice UI enhancement — customer names, item types, discount, billing period, record payment, overdue actions

**Status:** In Progress
**Assignee:** dev
**Created:** 2026-03-19
**Updated:** 2026-03-19

### Objective

Enhance the existing invoice frontend pages to close UI gaps vs. the base44 reference. The current pages are functional but missing several fields and user actions. This task depends on T-070 (API must be deployed first so new fields and endpoints are available).

#### Scope

**1. Invoice List page (`apps/web/src/routes/_authenticated/finance/invoices/index.tsx`)**

- Customer column: The API `GET /api/invoices` now returns `customer: { id, name }`. Display `customer.name` instead of the raw `customer_id`.
- Search: Extend the client-side filter to also match against `customer.name` (in addition to `invoice_number`). This mirrors base44's `InvoiceList` search behavior.
- Stats cards: Add a 4th stat — "Overdue" count, highlighted in rose-600. Base44 reference shows: All / Outstanding / Paid / Overdue.
- Row highlight: Rows with `status === 'OVERDUE'` already get `bg-rose-50/30`; keep this.

**2. Invoice Create page (`apps/web/src/routes/_authenticated/finance/invoices/create.tsx`)**

- Invoice date field: Add a `invoiceDate` date input (label: "Invoice Date", default: today). Maps to new API field.
- Billing period fields: Add `billingPeriodStart` and `billingPeriodEnd` date inputs (label: "Billing Period From" / "To"). Optional. Maps to new API fields.
- Discount field: In the Summary card, add a numeric input for discount (฿). Recalculate `total = subtotal + tax - discount`. Send `discount` to API.
- Item type: In the "Add Line Item" dialog, add a `item_type` Select with options: Rent (ค่าเช่า), Common Fee (ค่าสวนกลาง), Water (ค่าน้ำ), Electricity (ค่าไฟ), Parking (ค่าจอดรถ), Other (อื่นๆ). The value is stored in the line item's `item_type` field. Displayed in the line items table as a column between index and description.
- Contract auto-fill: When a contract is selected, offer to auto-add a "Rent" line item pre-filled with `contract.rent_amount`. Use a button "Auto-fill from contract" that adds a `{ item_type: 'Rent', description: 'ค่าเช่า', quantity: 1, unit_price: contract.rent_amount, amount: contract.rent_amount }` line item. Only show this button when a contract is selected and `contract.rent_amount > 0`.

**3. Invoice Edit page (`apps/web/src/routes/_authenticated/finance/invoices/$invoiceId/edit.tsx`)**

- Mirror all the same additions from the create page: invoice date, billing period, discount, item type column, contract auto-fill.
- Initialize form fields from the fetched invoice's new fields.

**4. Invoice Detail page (`apps/web/src/routes/_authenticated/finance/invoices/$invoiceId/index.tsx`)**

- Invoice info card: Display `invoice_date` (label: "Invoice Date"), `billing_period_start` / `billing_period_end` (label: "Billing Period"), `discount` (label: "Discount", only if > 0), `days_overdue` (label: "Days Overdue", only if > 0, in rose-600).
- Line items table: Add "Type" column displaying `item_type` when present.
- Summary totals: Show Subtotal / VAT / Discount (if > 0) / Total.
- "Record Payment" action: Add a "Record Payment" button in the page actions area (visible when status is SENT, PARTIAL, or OVERDUE). Clicking opens a dialog with fields: Amount (฿), Payment Date, Payment Method (dropdown: Cash, Bank Transfer, Cheque, Credit Card, Online), Reference Number (optional), Notes (optional). On submit, calls the existing `useCreatePayment` hook. On success, invalidates invoice detail and payments queries. The payments hook (`usePayments`) and API already exist — just add the UI trigger.
- "Mark as Overdue" action: Add a "Mark as Overdue" button (visible when status is SENT or PARTIAL and due_date is in the past). Calls `PATCH /api/invoices/:id/mark-overdue`.
- "Void Invoice" action: Add a "Void" button (visible when status is not PAID and not CANCELLED). Shows a confirmation dialog before calling `PATCH /api/invoices/:id/void`.

**5. Hooks (`apps/web/src/hooks/useInvoices.ts`)**

- Add `useMarkInvoiceOverdue(id)` mutation that calls `PATCH /api/invoices/:id/mark-overdue`.
- Add `useVoidInvoice(id)` mutation that calls `PATCH /api/invoices/:id/void`.
- Update `InvoiceQueryParams` in the hook to pass `search` string if needed (or keep client-side filtering — match existing pattern).

**6. `useCreatePayment` hook (`apps/web/src/hooks/usePayments.ts`)**

- Verify `useCreatePayment` mutation exists and invalidates both payment list and invoice detail queries on success. Add if missing.

#### Acceptance Criteria

- Invoice list shows customer names (not IDs) in the Customer column.
- Invoice list search matches both invoice number and customer name.
- Invoice list stats show All / Outstanding / Partial / Paid / Overdue counts.
- Invoice create/edit form includes: Invoice Date, Billing Period From/To, Discount, Item Type per line item, Contract auto-fill button.
- Invoice detail shows all new fields and the Record Payment dialog works end-to-end.
- Invoice detail "Mark as Overdue" and "Void" actions call the correct API endpoints.
- `pnpm build` passes with 0 TypeScript errors.

### Detail

#### Implementation Approach

- Add `apiPatch` to `apps/web/src/lib/api-client.ts` so PATCH calls can be made to mark-overdue and void endpoints.
- Update `apps/web/src/hooks/useInvoices.ts` with `useMarkInvoiceOverdue` and `useVoidInvoice` mutations.
- Invoice List: show `customer.name`, extend search to match customer name, add Overdue stat card (rose-600).
- Invoice Create/Edit: add invoice date, billing period start/end, discount, item_type select in line item dialog, and contract auto-fill button. Recalculate total = subtotal + tax - discount.
- Invoice Detail: show new fields (invoice_date, billing_period, discount, days_overdue), add Type column to line items, show discount/subtotal/VAT in summary, add Record Payment dialog, Mark as Overdue button, Void button with confirmation.
