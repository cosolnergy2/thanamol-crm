# T-088: Fix crypto.randomUUID crash on HTTP contexts

**Status:** In Progress
**Assignee:** dev
**Created:** 2026-04-07
**Updated:** 2026-04-07

### Objective

`crypto.randomUUID()` is a Web Crypto API method that is only available in secure contexts (HTTPS). The production CRM+FMS system runs on HTTP, causing a crash (`crypto.randomUUID is not a function`) on 7+ pages including Stock Issue, GRN, PR, PO, Vendor Quotation, Vendor Contract, and Vendor Invoice.

Fix by creating a shared `generateUUID()` utility in `packages/shared/src/utils/uuid.ts` that falls back to a manual UUID v4 implementation when the secure context API is unavailable, then replace all 40 direct `crypto.randomUUID()` calls across 13 files with this utility.

Acceptance criteria:
- `generateUUID()` utility exists in `packages/shared/src/utils/uuid.ts` and is exported from the shared package barrel
- Utility works on both HTTPS (delegates to `crypto.randomUUID`) and HTTP (manual UUID v4 fallback)
- All 40 direct `crypto.randomUUID()` calls in the 13 affected files are replaced with `generateUUID()`
- All 7 previously crashing pages load without error on HTTP
- `pnpm build` passes with zero errors
- `pnpm test` passes

### Detail

**Approach:**
- Created `packages/shared/src/utils/uuid.ts` with `generateUUID()` that delegates to `crypto.randomUUID()` when available (secure contexts) and falls back to a manual RFC 4122 UUID v4 built with `crypto.getRandomValues()` (available on HTTP).
- Exported `generateUUID` from the shared package barrel via `packages/shared/src/index.ts`.
- Replaced all 40 `crypto.randomUUID()` call sites across 13 files with `import { generateUUID } from '@thanamol/shared'` and `generateUUID()`.
- Wrote unit tests covering: valid UUID v4 format, uniqueness, delegation path, and fallback path.

**Files created:**
- `packages/shared/src/utils/uuid.ts`
- `packages/shared/src/utils/uuid.test.ts`

**Files modified:**
- `packages/shared/src/index.ts` — added `export * from './utils/uuid'`
- 13 route files under `apps/web/src/routes/_authenticated/facility-management/` — replaced `crypto.randomUUID()` with `generateUUID()`

**Doc references:**
- https://developer.mozilla.org/en-US/docs/Web/API/Crypto/randomUUID (secure context requirement)
- https://developer.mozilla.org/en-US/docs/Web/API/Crypto/getRandomValues (no secure context needed)
- https://www.rfc-editor.org/rfc/rfc4122 (UUID v4 bit layout)
