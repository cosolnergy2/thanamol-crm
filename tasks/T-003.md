# T-003: Seed data script (demo users + roles + sample data)

**Status:** Done
**Assignee:** dev
**Created:** 2026-03-15
**Updated:** 2026-03-15 (merged to dev, marked Done)

### Objective

Create a Prisma seed script that populates the database with realistic demo data so developers and testers can work without manually creating records, and so the dashboard renders meaningful content from day one.

**Seed data requirements:**

Roles (2):
- `Admin` ‚Äî full permissions across all modules
- `User` ‚Äî read/write permissions scoped to assigned resources

Users (3):
- `admin@thanamol.com` / `password123` ‚Äî assigned Admin role
- `user1@thanamol.com` / `password123` ‚Äî assigned User role
- `user2@thanamol.com` / `password123` ‚Äî assigned User role
- Passwords must be bcrypt-hashed (not stored in plain text)

Projects (2) with 10 units each:
- Project A: residential type, active status, 10 units of varying floor/building/type/price
- Project B: commercial type, active status, 10 units of varying floor/building/type/price

Customers (5):
- Mix of individual and corporate types
- Each with email, phone, address

Contacts (10):
- 2 contacts per customer, one marked is_primary

Companies (3):
- Different industries

TaskStatus records (4 default statuses):
- `New` (is_default: true, is_closed: false, order: 1, color: gray)
- `In Progress` (is_default: false, is_closed: false, order: 2, color: blue)
- `Review` (is_default: false, is_closed: false, order: 3, color: yellow)
- `Done` (is_default: false, is_closed: true, order: 4, color: green)

NotificationPreference records:
- One record per user for each notification type (task_assigned, deal_updated, payment_received)

**Script location:** `apps/api/prisma/seed.ts`

**Acceptance criteria:**
1. `npx prisma db seed` runs without errors and populates all data
2. Running the script twice is idempotent (uses upsert or checks for existence)
3. Can log in via API with `admin@thanamol.com` / `password123` and receive a JWT
4. Dashboard API responses include the seeded projects and customers

### Detail

**Implementation approach:**
- Create `apps/api/prisma/seed.ts` using Prisma Client with upsert operations for idempotency
- Hash passwords using `bcryptjs` (salt rounds: 10) before storing
- Seed in dependency order: Role ‚Üí User ‚Üí UserRole ‚Üí Project ‚Üí Unit ‚Üí Customer ‚Üí Contact ‚Üí Company ‚Üí TaskStatus ‚Üí NotificationPreference
- Add `"prisma": { "seed": "bun run prisma/seed.ts" }` to `apps/api/package.json`
- Write unit tests in `apps/api/prisma/seed.test.ts` validating data structures and exports
