# T-023: API — Quotation, CommercialQuotation CRUD + approval workflow

**Status:** Done
**Assignee:** dev
**Created:** 2026-03-15
**Updated:** 2026-03-15

### Objective
Build quotation management API with approval workflow for both standard and commercial quotations.

Quotation endpoints:
- GET /api/quotations — list with pagination, filter by status/customer/project
- GET /api/quotations/:id — with customer, project, unit, created_by relations
- POST /api/quotations — create
- PUT /api/quotations/:id — update
- DELETE /api/quotations/:id
- POST /api/quotations/:id/approve — approve quotation (requires manage_contracts permission)
- POST /api/quotations/:id/reject — reject quotation (requires manage_contracts permission)

CommercialQuotation endpoints (same CRUD pattern):
- GET /api/commercial-quotations
- GET /api/commercial-quotations/:id
- POST /api/commercial-quotations
- PUT /api/commercial-quotations/:id
- DELETE /api/commercial-quotations/:id
- POST /api/commercial-quotations/:id/approve
- POST /api/commercial-quotations/:id/reject

Acceptance criteria:
- approve/reject endpoints enforce manage_contracts permission via existing RBAC middleware
- Status transitions validated: only pending quotations can be approved/rejected
- All validation uses Elysia Box type system (t.Object()), not Zod
- Shared types for Quotation and CommercialQuotation in packages/shared/src/types/
- Rejection requires a reason field in the request body

Reference: QuotationList.jsx, QuotationCreate.jsx, QuotationDetail.jsx

### Detail

**Implementation approach:**
- Add `QuotationStatus` type and shared Quotation/CommercialQuotation types to `packages/shared/src/types/index.ts`
- Implement `apps/api/src/routes/quotations.ts` with full CRUD + approve/reject/pending endpoints using Elysia Box `t.Object()` validation and the `getUserAggregatedPermissions` RBAC guard for approve/reject
- Implement `apps/api/src/routes/commercial-quotations.ts` with same CRUD + approve/reject pattern (CommercialQuotation model has no `unit_id` or `approved_by` in schema, so approval just sets `status`)
- Auto-generate quotation numbers: `QT-YYYYMM-XXXX` for Quotation, `CQ-YYYYMM-XXXX` for CommercialQuotation — count existing records for the current month to determine the 4-digit sequence
- Register both route modules in `apps/api/src/index.ts`
- Write unit tests for both route files covering happy paths and error paths

**Status transition rules:**
- approve: current status must be SENT (treat as PENDING_APPROVAL equivalent per QuotationStatus enum) — the task says "only pending quotations can be approved/rejected"; looking at the schema `QuotationStatus` has DRAFT/SENT/APPROVED/REJECTED/EXPIRED. SENT is the pre-approval state.
- reject: same guard — status must be SENT
