# T-169: PO API + form overhaul

**Status:** Done
**Assignee:** dev
**Created:** 2026-04-01
**Updated:** 2026-04-01

### Objective

Overhaul the Purchase Order API and create form to expose all new schema fields added in T-154. Currently the PO form is missing PR linkage (which should auto-fill items), Company/Site/Unit scope, Vendor selection, PO Date, Payment Due Date, PO Type, Delivery Address, and the full Conditions and Documents sections. These gaps prevent POs from being properly tied to PRs and vendors.

Acceptance criteria:
- API create and update schemas accept `company_id`, `site_id`, `unit_id`, `po_date`, `payment_due_date`, `po_type`, `delivery_address`, `documents` (Json), `conditions` (Json)
- Optional PR link selector — when a PR is selected, item lines auto-populate from the PR's items
- Company / Site / Unit cascading selectors
- Vendor selection from the vendor list
- PO Date and Payment Due Date date-pickers
- PO Type dropdown sourced from `PO_TYPES` constant (Goods / Service / Rental)
- Delivery Address textarea
- Item lines include quantity, unit price, and total amount columns
- Conditions section mirrors the PR form (Payment Installment, WHT, VAT, Retention, Warranty, Credit Terms, Timeline, Late Penalty, Insurance)
- Documents upload (stored as Json array)
- All dropdown values sourced from constants in `packages/shared` — no hardcoded strings
